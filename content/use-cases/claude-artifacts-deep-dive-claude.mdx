## 情境

你問 Claude：「整個簡單嘅 landing page HTML 推我新產品」

冇 Artifact 嗰陣：

1. Claude 喺 chat 出 200 行 HTML
2. 你 copy 全部
3. 開文字編輯器，貼，存做 `index.html`
4. 雙撳開瀏覽器
5. Render 出嚟，2 個 layout 問題
6. 返 Claude，貼「呢度爛咗，呢段 fix」
7. Claude 出修改版
8. 重複 copy / 貼 / 存 / refresh

20 分鐘先見到最終版。每個小迭代 ~3-5 分鐘 overhead。

**有 Artifacts 之後**：

1. 你問 Claude，「呢度 → artifact 預覽」panel 即時 render
2. 你見到 HTML 即刻 work
3. 你話「H1 字大啲、CTA button vermillion」
4. Claude 直接更新 artifact，預覽即 refresh
5. 3 分鐘迭代週期

呢篇拆 5 個 Artifacts 真實用 use case，每個示範 prompt + 幾時用。

## 跟住做

### 1. 認識：咩會觸發 Artifacts

Claude 喺以下情況自動 render artifact：

- HTML / CSS / JS code → 即時預覽
- Python / JavaScript script → code + 可下載
- Mermaid diagram → 已 render 嘅 diagram
- React component → 互動預覽
- Markdown 長文（通常 > 1,000 字）→ 獨立文件
- SVG / 圖 / 圖標 → 已 render 嘅視覺
- 似 spreadsheet 嘅數據 → 表格

如 Claude 出咗 code 但冇 artifact 預覽，你可以明確講「render as artifact」/「show in artifact panel」。

### 2. Use Case 1 —— Landing page 樣板（HTML）

**Trigger prompt**：

```
整個簡單 1-page landing page 推 [產品]:
- 主 colour：vermillion #C42424 + cream #FBF7EA
- 結構：hero (大 headline + 1 sentence + CTA button) → 3 個 feature card → 1 個 testimonial → footer
- Font：serif 配 sans
- Mobile-responsive
- 用 inline CSS (single file)，唔好 external lib
```

Output：完整 HTML 喺 artifact panel，預覽 render 出嚟。

**迭代方式**：

```
H1 字細啲 (60px → 48px)，CTA button hover effect 加 darker shade，feature card 加大 spacing
```

Claude 直接更新 artifact，預覽即時 refresh。20 分鐘內多次迭代 → 最終起稿出嚟。

**限制**：Claude artifact 唔連真實 backend / form submission。真正 production deploy 仍要你 push 去 Vercel / Netlify。但樣板 / 概念示範 / pitch deck slide 已經夠。

### 3. Use Case 2 —— 圖表視覺化（HTML / SVG）

你有數據想快快視覺化，但唔想開 Excel：

**Trigger prompt**：

```
我 raw data:
- Q1 sales HK$120k
- Q2 sales HK$180k
- Q3 sales HK$140k
- Q4 sales HK$220k

請 render bar chart artifact:
- 主 color vermillion
- Background cream
- Y-axis HK$
- 上面 title「2025 quarterly sales」
- 數字 label on each bar
```

Output：live SVG 圖表 artifact。

**迭代**：

```
加 1 條 dashed reference line at HK$150k (target line)，label「target」
```

3 分鐘出可即用於演示嘅圖表。比 Excel 圖表 5 分鐘 setup 快。

### 4. Use Case 3 —— 互動原型（React component）

你係 design lead，想 demo 一個 UI concept 畀 stakeholder：

**Trigger prompt**：

```
整 1 個 React component:
- 個 form 同 3 個 input field: name / email / message
- Submit button (vermillion)
- 按下後 simulate API loading (1.5 sec) → success message animate in
- 用 Tailwind for styling
- 唔需要 real API call，淨係 visual demo
```

Output：互動 React component artifact。你可以喺 panel 入面真係撳 button + 見到 loading 動畫 + success 訊息。

**用途**：
- 設計 review 會議示範
- 畀 stakeholder 概念反饋
- A/B 試 2 個設計（重新 prompt + 整替代版本）

呢類互動原型比 Figma 靜態樣板 fidelity 更高，比真實 code 開發快 10 倍。

### 5. Use Case 4 —— 長文獨立文件

你草擬緊一份 8 頁嘅 client proposal：

**Trigger prompt**：

```
基於我 dump 嘅 client requirement (paste 你 brief)，draft 8-page consulting proposal:

結構：
1. Executive summary (1 頁)
2. 客 challenge understanding (1 頁)
3. Proposed approach + methodology (2 頁)
4. Timeline + milestone (1 頁)
5. Team + qualification (1 頁)
6. Investment + commercial term (1 頁)
7. Next step (0.5 頁)

語氣：professional, 唔好 corporate cliche, HK 客 audience。
```

Output：長文 artifact，markdown 格式。比 chat 入面 inline output 易讀，可以下載 / 分享。

**迭代**：

```
第 3 頁 approach section 加多 1 個具體 case study example
```

Artifact panel 即 update。最終版可 export markdown / PDF。

### 6. Use Case 5 —— 可下載嘅 code script

你想快快有 Python script 清理 CSV file：

**Trigger prompt**：

```
寫 Python script:
- Read input.csv (column: date, amount, category)
- 過濾 date >= 2025-01-01
- Group by category，sum amount
- Output result_cleaned.csv

用 pandas，要 error handling for missing column
```

Output：code artifact + download button。你直接下載 `script.py` 入 terminal 跑。

**迭代**：

```
加 progress bar (用 tqdm) 因為 input file 200k 行
```

Claude update artifact，新版即時可下載。

呢類快速 utility script 平時你 search Stack Overflow 30 分鐘 + 自己拼 code。Claude artifact：1 個 prompt + 1 次下載。

## 變化

### 變化 1：Multi-file project artifact

新版 Claude Code + 部分 Claude.ai 功能支援 multi-file artifact（整個 project 結構）：

**Trigger prompt**：
```
整個簡單 Next.js project:
- pages/index.tsx — landing
- components/Hero.tsx
- components/FeatureGrid.tsx
- styles/globals.css
- tailwind.config.js
- package.json
```

Output：多 file artifact，每個 file 獨立 tab。可下載做 zip。

你 unzip → `npm install` → `npm run dev` → 直接 run。

### 變化 2：用對話迭代 artifact

你唔滿意 artifact，反饋流程：

✅ **好嘅迭代 prompt**：
- 「Hero section 加 background image，用 cream / vermillion gradient」
- 「Feature card 由 4 個改 3 個，每個闊大啲」
- 「CTA button copy 改：『試 7 日』→『立刻開始』」

❌ **差嘅迭代 prompt**（容易令 Claude 混亂）：
- 「Make it better」（含糊）
- 「Like Apple website」（假設咗背景）
- 「Fix the styling」（冇講具體咩）

具體變化 > 模糊嘅改善。

### 變化 3：用 Artifact 教學 / 帶新同事

你想教同事一個概念（例如「點用 Excel pivot table」）：

**Trigger prompt**：
```
整一個 interactive tutorial HTML artifact:
- 上方解釋 pivot table 概念 (1 paragraph)
- 中間 show 1 個 example data (table render)
- 下方 step-by-step 點 build pivot table (5 step, numbered)
- 結尾 quiz 3 條 question with show-answer button
- 用 vanilla JS，唔好 framework
```

Output：完整教學 artifact。分享連結 / 出 HTML，同事自定節奏學。

呢類內部培訓素材平時要 video / Notion doc setup 1 個鐘，artifact 5 分鐘出。

## 一個心態

Artifacts 嘅深層價值唔係「render 預覽」—— 係**將迭代嘅阻力減到近零**。

當迭代週期由 5 分鐘 → 30 秒，**你嘅思維模式改變**：
- 冇 Artifact 嗰陣：你個腦預先 plan「完美 prompt」，怕 revise
- 有 Artifact 之後：你個腦自由咁「Try、see、adjust」快速嚟

呢個阻力減少對設計 / 創作 / 探索性工作影響顯著 —— 你試 5 個版本嘅成本低，所以你真係試 5 個（而唔係鎖實第一個 OK 嘅版本）。

優質 output 嘅根源唔係「prompt engineering」，係**「低阻力迭代令多元探索變得可能」**。

最後提醒：

- ✅ **迭代 > 完美**。Artifact 第一版 80% OK，迭代 3 次去到 95%。
- ⚠️ **Artifact ≠ production code**。樣板 / 示範 / 小工具 OK，但 production deploy 要真正測試 + edge case 處理。
- 🎯 **Artifact 可以下載 + 重用**。一次 build 嘅圖表模板 / form 結構，下次貼入新背景修改。

下次你想「整個簡單 HTML」嗰陣，唔好用 paste 出 text，直接畀 Claude render artifact。3 分鐘迭代週期 = 你工作嘅質素明顯提升。
