## 情境

你打工仔，每日同 Claude 傾 3-5 次嘢。

每次新 chat 開頭你做同樣動作：

```
「我係香港 marketing manager，做緊 [公司名] FMCG brand。
我哋 brand voice 係 friendly + professional + 偏 millennial。
我哋唔講「我哋 strive」呢類 corporate cliche。
今次想⋯⋯」
```

5 段 paste 完，先到正題。每次。每日 5 次。每星期 25 次。每年 1000+ 次 paste 同樣嘢。

如果有客問「依個 deck 寫得好行貨」，你 review 一次先發現 — 你 Claude 上次 chat 唔記得 paste brand voice，輸出已經漂咗。

呢個普遍問題嘅 root cause:

- Claude 每個 new chat = blank context
- 你個腦唔記得 paste 全部背景一致
- Context paste 5-10 段 = 起步成本高，少咗用

**Claude Projects 解決方法**：一個 project = 一個 context container。Attach reference file 一次，設定 system prompt 一次，往後 chat 入面自動帶住嗮全部。

呢篇 30 分鐘 setup，往後 6 個月每日慳 5-10 分鐘 paste 時間。

⚠️ Claude Projects 需要 Pro / Team / Max plan (US$20/月+)。Free plan 唔包。

## 跟住做

### 1. 開第一個 Project

登入 [claude.ai](https://claude.ai)，左 sidebar 揾到「Projects」 panel，撳 「+ Create project」。

填基本資料：

- **Project name**：用容易識別嘅名。例：「Bull Kicks Brand」/「個人寫作」/「客 X consulting」
- **Description**：1 句寫呢個 project 用嚟做咩
- **Privacy**：default 個人。如果係 Team plan + 想 share 同事就揀「Team」

撳 Create。

### 2. Set custom instructions (system prompt)

呢個係 project 嘅核心。每次 chat 都會自動帶住呢段 instruction，等如 Claude 永遠記得你嘅背景。

撳 project 入面嘅「Custom instructions」 (or「Project knowledge」 嗰 panel)，paste 你嘅固定 context：

```
你係幫我做 marketing copy 嘅 partner。

背景：
- 我係 香港 marketing manager
- 公司：[Bull Kicks - 鞋類 SME，eco / rescued 主題]
- Brand voice：誠實、唔賣弄、口語化、偶爾自嘲、唔用 corporate cliche
- 主要 audience：HK 25-40 歲 中產，識 sustainability 但唔 preachy

寫作規則：
- 唔寫「我哋 strive to」/「passionate about」/「best-in-class」
- 用具體 number 過 generic adjective (「賣 1,200 對」過「sold many pairs」)
- 中英 混 — HK 自然 mix，唔 force 翻譯 brand name
- 短句 multiple > long sentence singular

每次出 copy：
- 先問 1 條 clarifying question if 唔清楚 context
- 出 3 個 version，唔同 angle
- 結尾 1 句 honest assessment (邊個 version 你 think 最 strong + why)
```

呢段 system prompt 就係你 「每次 paste 5 段嘢」 嘅永遠代替品。

**Tip**: 第一次寫 system prompt 唔需要完美。用住先，第 2 個禮拜 iterate — 你會發現邊樣 instruction 你成日要再 paste，嗰啲就加返入去。

### 3. Upload reference files (Project knowledge)

撳「Add content」 / 「Upload files」 panel。Upload 嘅 file Claude 永遠可以 reference：

**Marketing project 例子**：
- `brand-guidelines.pdf` — 完整 visual + voice 規範
- `past-campaigns.md` — 過去 6 個 campaign 嘅 copy + 成效
- `competitor-snapshot.md` — 3 個競爭對手 brand voice 摘要
- `target-persona.md` — 2 個詳細 persona 文件

**個人寫作 project 例子**：
- 你過去 publish 過嘅 5 篇文章 (paste 入一個 MD file)
- 你最鍾意嘅作者嘅參考段落
- 你個人 「stop words / phrases」 list

**Consulting project 例子**：
- 客嘅 公司 overview deck
- 過去 3 個月 meeting note 整合
- 客 industry 背景 brief

**Upload limit (Pro plan)**: 通常 30-50 文件 / project, total size 限制 (check claude.ai 最新)。Format 接受 PDF / DOC / TXT / MD / image。

**最重要原則**：file 要 high-signal，唔好 dump 公司 wiki 全部入去。Claude 喺每次 conversation 嘅頭 inject 你 system prompt + reference file，太多 file = context window 食晒 + cost 上升。

精選 5-10 file 真係必要嘅，其他 link 喺 system prompt 入面提一句「如需更詳細 reference 我可以 paste」。

### 4. 試第一個 conversation

返 project 主頁，撳「+ New chat in [project name]」。

你第一個 prompt 可以好短：

```
幫我寫 IG carousel 5 slide，主題：8 月 promotion，舊鞋 50% off。
```

你會發現兩樣：
- Claude **自動** 用 你 brand voice (因為 system prompt)
- Claude **自動** reference 你 brand guidelines (因為 attached file)
- 你冇 paste 任何背景

呢個就係 setup 嘅價值。Conversation 5 段對話 → 5 個 IG post。Pre-Project setup：30 分鐘 paste + 30 分鐘 conversation。Post-Project：5 分鐘 conversation 就出結果。

### 5. Iterate weekly 提升 system prompt

頭兩星期你會發現 gap — Claude output 某啲嘢仲未 match 到期望。

每星期五 5 分鐘 ritual：

1. 打開 project
2. Review 過去 5 chat
3. 揾 邊 兩 三段 你手動改多次
4. 加嗰啲 rules 入 system prompt

例如：
- Week 1 你發現 Claude 成日寫 emoji 過多 → 加「Emoji 1-2 個 max per post」
- Week 2 你發現開頭 「I'd be happy to help!」 cringe → 加「直入 content，唔需要 helper intro」
- Week 3 你發現 CTA 重複「shop now」 → 加 5 個其他 CTA list

3 個月後你 project system prompt 自動進化到極準。

## 變化

### 變化 1：Multi-client consulting setup

你做 freelance consulting，5 個客同時 active：

開 **5 個獨立 Projects**，每個對應一個客：

- `Client A — Manufacturing` (industry context + 過去 deliverable)
- `Client B — Restaurant chain` (brand voice + menu reference)
- `Client C — Real estate` (regulation reference + market data)

每次 work session 入啱個 project 開 chat。你個腦轉 context 仲快 — project name 上面，你即時知 voice + reference 已啱。

**Anti-pattern**: 1 個「All clients」project + 每次 paste 「today is for Client X」 — context 容易撈亂，Claude 出錯 reference。

### 變化 2：Personal writing project (creator)

你寫 newsletter / blog / Substack：

System prompt：
```
你係幫我 polish writing 嘅 editor。

我 voice：
- 短句多
- HK 廣東話 + 英文 自然 mix
- 唔 self-help cliche，唔 too earnest
- 偶爾 self-deprecation OK
- 比較 over-formal，prefer conversational

點 help 我：
- 唔幫我 rewrite (我 own my voice)
- 幫我 spot：repetition / 模糊 sentence / 結構 weak point
- 用 inline suggestion (e.g.,「呢段 → 試 X」)
- 結尾 1 個 macro question (「呢篇 main argument 你想 reader 帶走咩？」)
```

Upload reference files：你過去最自豪嘅 5 篇大文 (各 paste 入 MD file) + 1 個 「我嘅 stop words list」。

新文章 draft → 丟入 project → Claude 用 你 voice 提具體 feedback。比 paste 入行貨 ChatGPT 強 10 倍。

### 變化 3：Team-shared project (Team / Enterprise plan)

公司 Team plan，5 人 marketing team：

開 shared project「Marketing — All」：
- Brand guideline file
- Voice 文件
- 舊 campaign reference
- 5 人都可以 access + 加 knowledge

每個 team member 喺呢個 project 開自己嘅 chat — 但 reference 一致，output voice 一致。

**注意**：shared project 嘅 system prompt 改動影響所有人。改之前 group chat 知會。

## 一個 mindset

Claude Projects 嘅 真正價值唔係「慳 prompt 時間」——係**外化你嘅隱性知識入可重用嘅 container**。

你腦入面有 5-10 樣 brand voice 規則自己都未講清楚過。寫 system prompt 嘅過程強迫你講清楚。講清楚嘅 knowledge = 可傳承畀 teammate / 未來嘅自己 / Claude。

3 個月後 你 system prompt 變成你 brand voice 嘅 1-page 精華版。新人入 team → 直接讀 project system prompt = onboarded。呢個額外好處比每日慳嘅 prompt 時間更有價值。

最後 reminder：

- ✅ **Project ≠ chat archive**。Project 係 context container，唔係 conversation 集合。
- ⚠️ **Upload file 唔好過 10 個**。Context window 食晒質素會跌。
- 🎯 **System prompt 每個禮拜 iterate**。第 4 個禮拜你會諗唔明點解 setup 前你死撐 paste。

下個 chat 立即 set up 第一個 project。1 個禮拜後睇返呢篇文，你會明點解 setup 一次值。
