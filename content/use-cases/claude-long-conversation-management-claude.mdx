## 情境

5 月 22 日，禮拜五 9pm。

你完結 3 個鐘嘅產品 brainstorm session 同 Claude。對話 thread 52 句：

- 第 1-15 句：你 dump 粗略產品 idea，Claude 反問釐清
- 第 16-30 句：refine target audience、brand positioning、USP
- 第 31-42 句：messaging 架構、key visual 方向
- 第 43-52 句：具體 launch copy、IG caption sample

呢個對話有優質 output，但：

- Claude 開始**慢** —— 每句要 30-60 秒先回
- 你問第 50 條問題，Claude「forget」第 8 句揀過嘅 brand voice 方向
- 你 scroll 上去揾返第 27 句嘅關鍵洞察 = 20 秒 scroll
- 你諗「下個禮拜重開條 chat 繼續 —— 但 chat 名 / context 我個腦唔記得」
- 你諗「想分享畀 partner 但 52 句唔可以叫佢全部讀」

呢個長對話嘅普遍問題：

- **Context window 食滿**：Claude 處理 conversation tokens 越多，回應越慢 + 質素漂移
- **記憶漂移**：Claude「記得」早期訊息但推理力減弱
- **發現摩擦**：你個腦知第 27 句有關鍵洞察，但揾嘅成本高
- **重用障礙**：1 個 chat = 1 個 silo，唔易重用

**解決框架**：

✅ 抽取對話嘅**價值**（洞察 / 決定）
✅ 壓縮入 Project knowledge base
✅ 重要 workflow 抽取做 Skills
✅ 關舊 chat，喺 Project 入面開新 chat

呢篇教你 5 步有系統咁拆長對話嘅程序。

## 跟住做

### Step 1：分辨對話嘅 value type

返 chat thread，scroll 過全部 52 句。每段訊息屬於邊類：

🎯 **A 類 —— 長駐 context（應入 Project knowledge）**
- Brand voice 規則
- 受眾 profile
- USP / positioning
- 將來 chat 永遠會用嘅背景

🔄 **B 類 —— 工序 / 程序（應變 Skill）**
- 點生成 IG caption（步驟）
- 點 polish email（規則）
- 重複任務嘅邏輯

📝 **C 類 —— 具體 output（應 archive 成 file）**
- 完成嘅 IG caption（10 條）
- 寫好嘅產品 brief
- 最終 visual 方向 note

🗑 **D 類 —— 廢話填充（可刪）**
- 「OK 試下」/「不錯」/「等等⋯」之類嘅 back-and-forth
- 你自己嘅情緒反應
- 死胡同方向（你之後 abandon 嗰啲）

每段訊息 tag 做 ABCD。20 分鐘有系統咁 review。

### Step 2：抽取 A 類 → Project knowledge

開新 Claude Project（按第 78 篇 setup）。

System prompt：
```
[A 類嘅長駐 context — 1 段整合]

例：
你係幫 Bull Kicks 做 marketing 嘅 partner.

Brand voice:
- 誠實 唔賣弄
- 自嘲 OK 但唔好 weak
- HK 廣東話 + 英文自然 mix

Target audience:
- HK 25-40 中產
- 識 sustainability concept
- 唔 like preachy / moralizing tone

USP:
- 「Rescued」sneakers 真實 second-life narrative
- 唔好「eco brand」行貨 positioning
- 具體 brand collab story
```

Upload 支援 file（`reference/brand-voice-decisions.md`）：
- 將 A 類嘅詳細訊息貼入一個 MD file
- 重要背景，Claude 永遠可以參考

### Step 3：抽取 B 類 → Skill

長對話入面你同 Claude 一齊發展出嘅程序（例如「點由 1 個產品 brief 生成 IG carousel」），抽取做可重用 Skill：

```markdown
---
name: ig-carousel-from-brief
description: Generate IG carousel (5 slide) from product brief. Trigger when user pastes a product brief + asks for carousel.
---

# IG Carousel Generator

## When to use
- User pastes product brief + asks IG carousel
- 唔 trigger for non-IG content

## Step-by-step
1. Read brief, extract: product hook / 3 features / target emotion
2. Plan 5-slide arc:
   - Slide 1: hook (大字 + 強 visual concept)
   - Slide 2-4: 3 features (1 per slide)
   - Slide 5: CTA + brand sign-off
3. For each slide:
   - Headline (max 8 字)
   - Body (max 25 字)
   - Visual concept (1 句)
4. 出 3 個 alternative tone (playful / serious / mysterious)
5. Recommend 1 + 1 句 why

## Rules
- 唔用「Game-changer」/「Revolutionary」/「Best-in-class」
- Each slide standalone readable (唔 require 跟住前一張)
- 中英 mix 自然
```

呢個 skill 由你 52 句對話抽取 —— 但變得可攜可重用，喺任何將來 chat 都可以叫起。

### Step 4：Archive C 類 → 外部 file

具體 output（C 類）儲做外部文件：

```
/Bull-Kicks-Launch-Project/
  ├── 2026-05-22_brand-voice-decisions.md
  ├── 2026-05-22_product-brief-final.md
  ├── 2026-05-22_ig-captions-10.md
  └── 2026-05-22_visual-direction-notes.md
```

放 Notion / Google Drive / GitHub 視乎 team。

呢類 archive 比 chat thread search 易揾 —— file name 直接講明係咩。

### Step 5：喺 Project 入面開新 chat

關舊 52 句 thread（或者 archive —— 唔 active 就 OK）。

返 Project，「+ New chat」開新 thread。

第一句訊息可以好短：
```
基於我哋上次 brand voice + product brief，generate Mother's Day campaign IG caption 5 個。
```

Claude 即時用 Project 嘅 brand voice + 可能叫起 ig-carousel-from-brief skill。

5 分鐘出 5 條 caption，全部 voice 一致。**vs 重貼 5 段背景**。

每星期你喺 Project 入面開新 chat。Chat thread 持續短 + 精準。

## 變化

### 變化 1：「我未 setup Project」—— 緊急匯出法

仲未 build Project / Skill，但條長 chat 已經關鍵：

**快速匯出方法**：

1. 開 Claude chat
2. 滾上頂
3. ⌘+A 全選
4. Copy
5. 貼入 markdown file（例如 `conversation-2026-05-22.md`）

存入你嘅 archive folder。下次用嘅時候打開該文件，參考具體決定「per our 5/22 conversation, brand voice OK」。

呢個係手動 band-aid，唔有系統 —— 但緊急場景 OK。

### 變化 2：將對話交接畀同事

你離職 / 換 project / 同事接手，要交接複雜 Claude 對話：

**交接文件結構**：

```markdown
# Project [Name] — Claude Conversation Handoff

## What I've been using Claude for
- [Daily / weekly task list]

## Key context Claude knows
- [Brand voice rules — paste]
- [Target audience — paste]
- [USP — paste]

## Patterns I've established
- [Specific prompt patterns that work]
- [Things that don't work]

## Setup
- Claude Project name: [link if shared Team plan]
- Skills installed: [list]
- Reference files location: [link]

## Suggested onboarding
1. Read this doc
2. Open Project, review system prompt
3. Try 1 simple chat to see voice
4. Iterate system prompt as needed
```

呢個 doc 比「開 ChatGPT 自己摸」幫同事慳 3-6 個禮拜 ramp 時間。

### 變化 3：對話變內部培訓素材

長 chat 入面有「教學 moment」—— Claude 解釋某個概念你第一次明白。抽取做培訓素材：

1. 識別「教學 moment」訊息
2. Copy 嗰段 Q + A
3. 排入培訓文檔
4. 加 screenshot / 圖示

例：你問「點評估 Schengen 簽證被拒風險」，Claude 出 5 因素框架。抽取呢段做 internal sales 培訓素材 → 新同事 5 分鐘讀 = 慳 1 個鐘 onboarding。

## 一個 mindset

長對話嘅悖論：**對話越長越有價值，越難維持質素 + 重用**。

呢個悖論嘅原因：chat thread 係 ephemeral 媒介（訊息嘅順序），但有價值內容嘅本質係結構化 + 可搜索 + 可重用。錯配直接令人 frustrate。

呢個 mindset 嘅 reframe：chat thread 唔係「永久 archive」—— chat thread 係「探索 + 迭代嘅 sandbox」。結晶化嘅 output 應該遷移到結構化容器（Project / Skill / file）。

呢個 process 同你寫 journal 一樣：
- Journal 條目（自由思考）—— 等於 chat
- 個人 essay（結晶化思考）—— 等於 Project knowledge / Skill

兩種都有價值，唔好混淆。

最後提醒：

- ✅ **每星期 1 個鐘 review 長 chat → 抽取價值 → 遷移**。比「我下次 scroll 揾」可靠。
- ⚠️ **50+ 句 chat 真係慢**。唔好硬撐落去。喺 Project 入面開新 thread + 參考過往 output。
- 🎯 **抽取嘅功夫第 1 次要 1 個鐘，但 setup 之後每月慳 5-10 個鐘**。回報明顯。

下個禮拜抽 1 個鐘，拆解你最長嘅活躍 chat。你會 surprise —— 50% 內容屬於 D 類（廢話），真正有價值嘅 A + B 其實只係 20%。20% 嘅抽取 + 結構化 = 高槓桿嘅時間投資。
