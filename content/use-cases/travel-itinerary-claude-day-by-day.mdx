## 情境

你哋情侶 / 一家大細 / 一班朋友 plan 緊去日本 / 韓國 / 東南亞 / 歐洲 7-14 日。你已經買好機票 + 訂好酒店，淨低「每日去邊」嘅行程未排。

你打開 Google Sheet，準備列出每日計劃。睇吓 Klook 推介、blog 文章、Instagram 收藏——

**4 個鐘之後，你仲係留喺第一日。** 因為：

- 唔知第一日應該預留時間適應時差（去近啲、唔好行太多路）定係進取啲（趁精神去玩）
- 每個地方喺 Google Maps 距離唔同，但你冇方向感
- Klook 推介 25 個地方，但 7 日入面邊個最適合
- 旅遊 blog 寫得好詳細，但「我呢一班人」唔知啱去邊幾個景點

呢篇教你用 Claude **由你嘅風格、節奏、預算、同行組合**，**出一個你可以直接跟住行**嘅逐日行程——連埋 Google Maps 驗證同「落雨後備方案」。

## 跟住做

### 1. 先誠實評估你嘅「旅行哲學」

呢一步 90% 嘅人會跳過，但佢係度出實用行程嘅關鍵輸入。

問吓自己：

- **節奏 (Pace)**：每日 3-4 個景點（chill）vs 6-8 個景點（行程緊湊）vs 純休息 + 1 個主要景點
- **風格 (Vibe)**：遊客必去 vs 地道隱世景點 vs 純粹為食 + 行 cafe
- **每日活躍時間**：早起型 vs 瞓到自然醒型
- **同行組合**：單獨 / 情侶 / 有小朋友嘅家庭 / 4 個朋友一齊去（每種組合影響好大）
- **預算級別**：每人每日預算（食 + 活動 + 交通，唔包酒店）
- **行路耐力**：每日行 4 個鐘以上都 OK vs 想最多行 2 個鐘
- **天氣應變**：1-2 日落雨嘅後備活動

### 2. 用呢個 prompt

<PromptBlock label="完整 prompt — 7-14 日逐日行程">
我哋 [人數 + 同行組合，例：情侶 / 一家四口 2 個小朋友 8 同 12 歲 / 3 個朋友] 去 [目的地] [天數] 日。

物流安排：
- 機票：[到達機場 + 時間]、[離開機場 + 時間]
- 酒店：[城市 + 邊區 + check-in/check-out 日期]
- 已 book 嘅活動：[列出任何確實 book 咗嘅嘢，例：USJ 門票 Day 4 / 環球影城 Day 5]

旅行哲學：
- 節奏：[每日 3-4 個景點 / 每日 6-8 個景點]
- 風格：[遊客路線 / 地道 / 搵食為主 / 混合]
- 每日活躍時間：[早起 / 瞓到自然醒 / 朝早 8 點起]
- 行路耐力：[每日行 6 公里以上 OK / 最多 4 公里 / 越少越好]
- 每人每日預算（食+活動+交通，唔包酒店）：HKD [數字]
- 天氣應變：[後備方案想多啲室內活動 / 戶外後備方案都 OK]

特別偏好 / 避忌：
- 我哋鍾意：[列 3 樣]
- 我哋唔鍾意：[列 3 樣，例：「唔鍾意行大型 outlet」]

請整逐日 (day-by-day) 行程，每日結構：

**Day X：[一句話總結主題]**
- **朝早**（XX:XX–XX:XX）：[活動 + Google Maps 地名 + 距離起點幾遠]
- **午餐**（XX:XX）：[餐廳 + 菜式 + 預計花費 + 需要預約嗎？]
- **晏晝**（XX:XX–XX:XX）：[活動 + 由食飯地方點去 + 交通時間]
- **晚餐**（XX:XX）：[餐廳 + 菜式 + 花費]
- **夜晚**（XX:XX–XX:XX，可選）：[輕鬆活動]
- **全日大約步行距離**：~X 公里
- **預計每人總花費**：HKD ~XXX
- **落雨後備方案**：[替代嘅室內活動]

**主題關聯**：解釋點解呢日係呢個主題（適應時差 / 地道探索 / 一日遊 / 等等）

**附上最後總結：**

- 預計總花費（每人）
- 每日行路 + 搭車總時間
- 提我可以預先 book 嘅活動（熱門要霸位嗰啲）
- 3 個「我可能想換走」嘅景點（提供替代方案）

語氣：具體、可採取行動、唔好誇大（唔好用「最適合打卡」呢類行銷字眼）。
</PromptBlock>

### 3. 用 Google Maps 驗證

Claude 嘅輸出 90-95% 啱，但具體地點嘅準確度偶爾會出錯（無中生有）。
**關鍵驗證**：

- 每個地點喺 Google Maps 搜尋一次，確認真係存在 + 地址啱
- 距離 / 交通時間用 Google Maps 嘅「路線」驗證
- 餐廳如果需要預約，一定要查證（Tabelog / OpenTable / 餐廳官網）

15 分鐘嘅驗證工作可以避免你到咗當地先發現「呢間餐廳上年執咗」。

### 4. 適度「留白」

完美塞滿嘅行程永遠都會出事。預留「無計劃時間」：

- 每日朝早 / 晏晝留一節「空檔」等你隨心發揮
- 中間 1 日完全唔排行程，純粹「休息 + 亂行」
- 飛機 / 火車延誤緩衝：落機後唔好即刻排主要景點

呢類「留白」唔係偷懶，係為咗保留旅行嘅樂趣。

### 變化 2：短線遊回報最大化（3-4 日）

<PromptBlock label="變化 2 — 短線遊回報最大化">
我哋只有 3-4 日嘅短線旅行去 [目的地]。
時間有限，要將單一「代表性體驗」最大化。

請：
1. 揀一個城市 / 地區集中玩，唔好分散
2. Day 1 安排適應時差 + 標誌性必去景點

- 節奏：偏慢，唔趕景點

## 一個思維轉變

旅行行程最大嘅迷思：「完美計劃等於完美旅程」。

事實係：**最記得嘅旅行時刻通常係計劃之外嘅嘢**——意想不到嘅繞道、同當地人傾偈、隨意搵到嘅 cafe、突然落雨改去博物館。

Claude 嘅用處唔係幫你「編死晒」，係令你有個底：**「我有一個合理嘅計劃，所以我可以充滿自信咁偏離原本路線」**。

冇計劃 vs 有計劃再改：兩個最終都會改變路線，但**後者你唔會有焦慮**，亦保留咗改計劃嘅自覺（每日改計劃前你知道有咩取捨）。

最後提醒：**輸入嘅質素決定輸出**。你誠實描述你哋班人嘅互動情況（包括取捨：你伴侶鍾意 shopping 但你好討厭），出嚟嘅行程就會啱晒你哋。你用「標準典型遊客」嚟形容自己，個輸出就會係行貨。呢個唔係 AI 嘅限制，係輸入嘅限制。
