## 情境

星期一早上 8:45。你坐返公司 / 開電腦 work from home。打開 Gmail：

> **247 unread, 35 from this weekend**

你開始 scroll，但 brain 仲未 boot up。前 15 分鐘係 scroll、scroll、scroll，5 個 email 開咗冇覆、3 個 issue 你 register 唔到 priority。 9:30，meeting 開始，你還未真正 start 你嘅 day。

呢個 weekly pattern 每個禮拜 lose 30-45 分鐘。 一年 = 26 小時 = 3 個 work day。

呢篇 setup Cowork 每週一早 9am 自動 run：

- Triage 過去 3 日所有 email
- 分類（actionable / FYI / spam / waiting on me / waiting on others）
- Draft 3-5 個你應該即刻 reply 嘅 email
- 寄一封 「Inbox Brief」email 落你個人 inbox

你 9am 開電腦，個 brief 已經 sitting there。 你睇 5 分鐘，撳 send 3-4 個 draft，9:15 你 inbox zero、 brain ready，準備 meeting。

> Caveat：Cowork 直接 read 你 Gmail，需要 OAuth permission。你 set 之前確認公司 policy，敏感行業（finance / healthcare / law）可能唔可以。**呢個 process 不可以 send email 自動——只可以 draft + 等你 review**。

## 跟住做

### 1. 整理 Gmail label 結構

Cowork 嘅 triage 質素 depend 你 label setup 嘅清晰度。 喺 setup 之前喺 Gmail 整：

```
01-Action          ← 我需要 reply / take action
02-Awaiting        ← 我已 reply，等對方
03-Reference       ← 唔需要 action，但要 keep
04-Newsletter      ← Subscribe 嘢
05-Calendar        ← Meeting invite
99-Archive         ← Done, 可以 archive
```

如果你已經有 Gmail label，用你自己嘅 schema。Cowork 識 learn 你嘅 system。

### 2. 用呢個 prompt setup

<PromptBlock label="完整 prompt — Setup 每週一早 inbox triage">
我想 set up 一個每週一早 9am 自動執行嘅 inbox triage task。流程：

**Trigger time：** 每星期一 09:00（HKT）

**Input source：**
- 我嘅 Gmail（OAuth-connected）
- 我自己 label structure：[paste your label list]

**每次 run：**

**第一步：collect email**
- Scan past 3 days（Friday 9am → Monday 9am）
- 排除：
  - 已讀但 archived 嘅
  - 我已經 reply 嘅 thread
  - Marketing email（用 List-Unsubscribe header 識別）
  - Calendar invite（separately handle）

**第二步：classify**
每封 email 分入以下 bucket：

A. **Action — high priority**：要我 reply / decide 嘅，sender 係 senior 或 customer 或 deadline-sensitive
B. **Action — normal**：要我 reply，但唔 urgent
C. **FYI — keep informed**：唔需要 reply，但內容對我 future 有 implication
D. **Awaiting me**：我之前承諾 follow-up 嘅 thread，對方等緊
E. **Awaiting others**：我已 ball back to 對方，等對方
F. **Noise**：marketing / newsletter / spam
G. **Calendar**：meeting invite / reschedule request

每封 email：「from + subject + 你嘅 classification + 1 句 rationale」

**第三步：identify hidden importance**
特別 flag 以下 「easy to miss」 嘅 email：
- 我被 @ mention 但係 part of 大 thread
- Subject line 唔搶眼但內容係 deadline 或 ask
- 由 senior（VP+、CEO、important client）send 但短嘅 email
- 任何 reply 喺一個我已經 「forgotten」 嘅舊 thread

**第四步：draft replies for 3-5 emails**

揀 3-5 個最 critical 嘅 actionable email，draft reply。要求：

- Draft 唔好 verbose，2-3 句到位（保留我自己 tone preference）
- 如果係 complex decision 唔可以一句答，draft holding pattern：「Received, will revert by [reasonable time]」
- 如果係 ask 我提供 info 我 not yet have，draft「Acknowledging the ask, will follow up by [日期] with [specific deliverable]」

每個 draft：
- Email recipient
- Original email summary（1 句）
- Draft reply text
- 我可以 send as-is 嘅 indicator（「Send as-is」） vs 需要 edit（「Draft for your customization」）

**第五步：寄 Inbox Brief 落我**

Email 我 subject：「Monday Inbox Brief — X actionable, Y awaiting you」

Body 結構：
1. **Headline numbers**：total processed、A+B count、urgent flags
2. **High priority action list**（按 priority 排）
3. **Draft replies**（above）— 用 markdown code block format，方便我直接 copy paste 落 Gmail
4. **Hidden importance flags**（容易 miss 嗰啲）
5. **Awaiting others — overdue list**（我 ball-backed 但對方 > 5 日冇覆嘅，可能要 follow-up）
6. **FYI digest**（5 個 most important FYI 嘅 1-句 summary）
7. **Noise filtered out**：count + 任何 unusual marketing pattern

**重要 constraints：**
- 唔好 send 任何 email 自動（all draft only）
- 唔好 archive / delete / mark read 任何 email（只 read）
- 如果 thread 比較 complex，stay safe，標 「Manual review needed」 唔好 draft
- Privacy：唔好 forward 任何 personal email 內容俾我之外嘅人

Setup 好之後 confirm，再 schedule 落每星期一 9am 行。
</PromptBlock>

### 3. 第一週 dry run

第一次 set up 一定要 dry run：「即時跑一次（now，唔係下個星期一），showcase output，唔好真 send 個 brief」。

第一次見到 output 嗰陣 check：
- Classification accuracy（A vs B vs C 嘅判斷準唔準）
- Hidden importance：有冇 catch 到啲 subtle ask
- Draft reply：tone 似唔似你

執 fine-tune（你會發現某 sender 嘅 weighting 要調、某 keyword 嘅 false positive 等）。 通常頭兩個禮拜會 iterate，第三週開始 stable。

### 4. 第三週開始嘅你

每星期一 9am 開機，individual mental cost：

- 開 Inbox Brief（5 分鐘 read）
- Copy-paste 3 個 「Send as-is」 draft + 撳 send（3 分鐘）
- 1-2 個 「Draft for customization」 fine-tune 同 send（5 分鐘）
- Acknowledge `Hidden importance flag`（2 分鐘）
- 9:15 done，inbox 處理咗 70%，brain ready 開 meeting

呢個 leverage 唔係慳時間，係 **降低 anxiety**。 你唔再驚 「我可能 miss 咗一封重要 email」——Cowork 已經 review 過。

## 變化例子

### 變化 1：每日早上 8:30 跑（vs 每週一）

<PromptBlock label="變化 1 — 每日早上 8:30 run">
我 email volume 高（200+/day），每週一 run 唔夠 frequent。

請 modify 為：
- Trigger：每日早上 08:30
- Scope：past 24 hours email
- Format simplified：1 段 summary + 3 個 draft（唔需要 elaborate sections）
- Friday 嗰個 run 加 weekend coverage hint：「有冇 weekend 跟進預期？」

呢個 cadence 適合 customer-facing role、客戶 expectation real-time。
</PromptBlock>

### 變化 2：加 task management integration

<PromptBlock label="變化 2 — Email → Todo">
我用 [Todoist / Things / Asana] manage tasks。

加 features：
- Cowork triage 完之後，將「Action」bucket 嘅 email 自動 create 對應 task
- Task title：「Reply: [email subject]」
- Task due：reasonable timeline（urgent = today, normal = within 3 days）
- 唔重覆 create task（用 email id 做 unique key）

如果 trade-off，呢個 process 比上面複雜啲，需要 third-party tool integration。
</PromptBlock>

### 變化 3：你係 manager，要 monitor team email volume

<PromptBlock label="變化 3 — Manager dashboard">
我 manage 一個 5-person team。
我自己有 weekly inbox triage 已經做緊。

加 feature：team-level pattern monitoring
- 每週一 10am（你自己 brief 之後 1 小時）寄 team-level email：
  - 每個 team member 收到嘅 email volume（趨勢）
  - Flag：有冇 team member 收到大量 escalation
  - Flag：有冇 external sender 對 multiple team member 嘅 outreach（potential conflict 信號）

Privacy 嚴格：唔睇 individual email content，只睇 metadata（sender + 數量）。 任何 individual 嘅 detail 要 manager 自己問 team member。
</PromptBlock>

## 一個 framing

Email triage 嘅問題唔係 「我 inbox 太多」——係 **「我 brain 用嚟掃 inbox 之後，已經 tired 喺真正 work」**。

每個禮拜，你嘅 sharpest brain time（早上 9-11am）用嚟做 機械式 sorting。 呢個 根本係 resource misallocation。

Cowork 嘅 leverage：**將 機械式 工 shift 落 9am 之前 + 自動，將你嘅 sharp time 留落 high-judgment work**。

呢個 setup 真正嘅 ROI 唔係慳嗰 45 分鐘——係**你 monday morning 嘅 1-on-1 / strategy meeting，可以 fresh、prepared、 thoughtful**，唔係 「我未睇 email 我都唔知乜事」。

最重要嘅 reminder：**永遠唔好讓 Cowork 自動 send email**。Draft + Human review + Send，呢個三步永遠 keep human accountability。
