## 情境

你係細店老闆。每月 1 號 routine：

- 9am：開 Stripe / Shopify dashboard，export 上月 transaction CSV
- 10am：import 入 Google Sheet，整理 column
- 11am：pivot table 整 category breakdown
- 12pm：寫 1 頁 summary 畀 partner / 會計
- 1pm：format + send

4 個鐘月底儀式，每月準時上演。你試過：

- 「Excel template 自動」：cleanup 之後跑 —— 但每月數據略有不同，template 唔啱
- 「Bookkeeping app」：收月費，出嘅報告太行貨
- 「會計整」：$$ 高 + 你仍然要覆檢

呢個問題嘅本質：

- 數據 80% 一致（column 結構一樣）
- 洞察 20% 變（每月 highlight 唔同）
- AI 應該處理嗰 20% 變嘅部分

呢篇用 Claude + Apps Script 整合，1 個鐘 setup，往後每月報表撳 1 下就行。

⚠️ 呢個 use case 假設你會用 Google Sheets + 識讀基本 JavaScript（Apps Script 就係 JS）。完全唔識 code 嘅用家，跳到「變化 1」用手動貼 flow。

## 跟住做

### 1. Prep 你 Sheet 結構

打開 Google Sheet，setup 兩個 tab：

**Tab 1：`raw-data`**
- A 欄：日期
- B 欄：金額
- C 欄：客戶
- D 欄：產品 / 服務
- E 欄：渠道（online / instore / wholesale）
- F 欄：備註

貼上月交易入呢個 tab。

**Tab 2：`monthly-report`**
- A1：「Monthly Report —」+ B1：月份
- A3：Claude 出嘅位

### 2. 拎 Anthropic API key

去 [console.anthropic.com](https://console.anthropic.com)：

1. Sign up / login
2. Settings → API Keys
3. Create key → label「sheets-integration」
4. **Copy 嘅 key 即刻儲好**（只見得到一次）

API 用量收 metered fee：1 個 monthly report ≈ HK$0.5-2，月底快速自動化嘅成本好平。

### 3. 寫 Apps Script

返 Google Sheet → Extensions → Apps Script（開新 editor tab）。

貼呢段 code：

```javascript
function generateMonthlyReport() {
  const API_KEY = PropertiesService.getScriptProperties().getProperty('ANTHROPIC_API_KEY');
  if (!API_KEY) {
    SpreadsheetApp.getUi().alert('Set ANTHROPIC_API_KEY in Script Properties first');
    return;
  }

  // Read raw data from Tab 1
  const rawSheet = SpreadsheetApp.getActive().getSheetByName('raw-data');
  const data = rawSheet.getDataRange().getValues();
  const headers = data[0];
  const rows = data.slice(1);

  // Convert to text format for Claude
  const dataText = rows.map(row =>
    headers.map((h, i) => `${h}: ${row[i]}`).join(' | ')
  ).join('\n');

  // Build prompt
  const prompt = `你係 monthly sales 報表 analyst. 我嘅 raw transaction data：

${dataText}

請整 1 頁 monthly report. 結構：

📊 **EXECUTIVE SUMMARY (2-3 句)**
- 上月 total revenue
- 1 句 trend (vs typical month)
- 1 個 highlight + 1 個 concern

📈 **REVENUE BREAKDOWN**
- By channel (online / instore / wholesale): amount + % share
- By product category: top 5 amount + % share
- By customer type (new vs returning): split

⚠️ **NOTABLE PATTERN**
- 任何 unusual transaction (refund / large outlier)
- 任何 trend worth flag (specific customer 增加, product 突然 grow)
- 任何 risk signal

🎯 **NEXT MONTH FOCUS (1-2 個 actionable)**
- 具體建議 based on data
- 唔好行貨「improve marketing」

語氣：data-driven, 唔好 fluffy. 出 markdown format.`;

  // Call Anthropic API
  const response = UrlFetchApp.fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'x-api-key': API_KEY,
      'anthropic-version': '2023-06-01',
      'content-type': 'application/json'
    },
    payload: JSON.stringify({
      model: 'claude-opus-4-7',
      max_tokens: 2000,
      messages: [{ role: 'user', content: prompt }]
    })
  });

  const result = JSON.parse(response.getContentText());
  const reportText = result.content[0].text;

  // Write to Tab 2
  const reportSheet = SpreadsheetApp.getActive().getSheetByName('monthly-report');
  reportSheet.getRange('A3').setValue(reportText);

  SpreadsheetApp.getUi().alert('Report generated! See monthly-report tab.');
}
```

### 4. 設定 API key（安全）

Apps Script editor → Project Settings → Script Properties → Add property：

- Name：`ANTHROPIC_API_KEY`
- Value：貼你 API key

Save。

呢個方法比直接 hardcode 入 code 安全 —— script 唔會透過分享 / export 漏 key。

### 5. 加 menu trigger（撳 1 下就行）

返 Apps Script editor，加多一段 code：

```javascript
function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu('Monthly Report')
    .addItem('Generate', 'generateMonthlyReport')
    .addToUi();
}
```

Save。Reload Google Sheet。

你會見到頂部多咗一個 menu「Monthly Report」→「Generate」。撳一次 → 30-60 秒 → 報表出咗。

### 6. 第一次跑 + 迭代 prompt

第一次 run —— 你會發現邊樣 output 唔啱：

- 某個 dimension 唔夠 detail → 改 prompt
- 語氣過於 corporate → 加語氣規則
- Format 錯 → 將 format 講清楚啲

每次改完 prompt re-run，5 次 iteration 後 prompt 會收斂到你要嘅質素。

呢個 1 個鐘 setup，往後每月底撳 1 下跑 30-60 秒，慳 3.5 個鐘。1 年慳 42 個鐘。

## 變化

### 變化 1：唔識 code —— 手動貼 flow

冇 Apps Script setup，每月手動 5 分鐘貼 flow：

1. Sheet → File → Download → CSV
2. 開 CSV → Copy 全部 transaction text
3. 開 Claude Project（按第 78 篇 setup）
4. 貼數據 + 一個長駐 prompt：

```
新一月 transaction. 用 our standard report template (in project knowledge) 生成.
```

5. Copy output → 貼入 Sheet 嘅 monthly-report tab

20 分鐘 / 月（vs 手動 4 個鐘）。冇 setup 負擔。

取捨：每月仍要你手動啟動。Apps Script 一次 setup 就永遠自動。

### 變化 2：Slack 整合 —— 自動將報告 send 落 channel

加 Slack webhook 整合，每月 1 號 9am 自動觸發 + 將報告 send 落 #monthly-reports channel：

```javascript
function sendToSlack(reportText) {
  const SLACK_WEBHOOK = PropertiesService.getScriptProperties()
    .getProperty('SLACK_WEBHOOK_URL');

  UrlFetchApp.fetch(SLACK_WEBHOOK, {
    method: 'POST',
    contentType: 'application/json',
    payload: JSON.stringify({
      text: `📊 Monthly Report — ${new Date().toLocaleDateString()}\n\n${reportText}`
    })
  });
}
```

時間 trigger：Apps Script editor → Triggers → Add trigger → `generateMonthlyReport` → Time-driven → Monthly → Day 1, 9-10am。

每月 1 號自動 run + send Slack。完全 hands-off。

### 變化 3：擴展去其他報表（每週 / 每季 / 每年）

同樣框架，唔同節奏：

- **每週**：星期一 9am trigger，集中每週 KPI 變化
- **每季**：4/7/10/1 月 1 號 trigger，跨月匯總，按年比較
- **每年**：1 月 1 號 trigger，全年回顧 + 下年計劃

每個節奏開獨立 menu 項目 + 自己嘅 prompt 模板。

進階：出**儀表板**（Apps Script 都可以基於 Claude output 整圖表），同時 send email + Slack。

## 一個心態

「自動化」呢個詞容易被 oversell。實際 frame：**自動化 ROI 嘅計算唔係「節省 100% 時間」，係「節省重複嘅 80% 時間，保留 20% 判斷留畀你」**。

每月底你 4 個鐘：
- 3.5 個鐘 = 重複 pattern（export / 清理 / pivot / 排版）
- 0.5 個鐘 = 真正判斷（邊個 trend 顯著 / 邊個 concern 真係值得 flag）

呢個 setup 自動化 3.5 個鐘。0.5 個鐘判斷仍然你做 —— 你讀 Claude draft + 加你自己嘅背景洞察（例如「上月 client X 落大 order 因為一次 event，唔可持續」）。

呢個心態嘅 reframe：自動化唔係取代你 —— 自動化釋放你嘅判斷時間。

最後提醒：

- ✅ **API 成本低**：1 份每月報告 ≈ HK$0.5-2，1 年 < HK$30。比「人工 4 個鐘 × 12 個月 × HK$200/hr = HK$9,600」係兩個世界。
- ⚠️ **API key 永遠唔好 commit Git / 分享**。用 Script Properties / 環境變數。
- 🎯 **第 1 個月你 prompt 唔完美**。每月 5 分鐘調 prompt，第 4 個月已經夠精準。

下個月 1 號試撳 1 下行。慳 3.5 個鐘。Compound 12 個月。
