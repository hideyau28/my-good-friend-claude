## 情境

你發現 Claude Code 一個禮拜前，睇過 Anthropic blog post + 兩個 dev Twitter thread。

你試吓 install:

1. Google「install claude code」 → 第 1 結果一個 npm package 名撞咗
2. 揾 official docs → install command 但你唔清楚邊個 directory run
3. Run 完，error message：「ANTHROPIC_API_KEY not set」
4. 揾 API key page → create key → 唔清楚喺邊 set env var
5. Set 完，試 run 第一個 command → permission error
6. 卡住 1 小時，放棄

呢個 install 死結：

- **Docs 預設咗對開發者嘅「基本認知」**: env var, shell config, npm permission, etc
- **Stack Overflow 答案已經過時** (Claude Code 係新 tool, 仲 evolving)
- **頭一個鐘卡住 = 永遠放棄** — 你唔會再試

呢篇就明確防止呢樣嘢：30 分鐘有系統 install，覆蓋 Mac + Linux，每 step 解釋「點解」同「如果出錯」。完成後你 run 第一個 Claude Code session。

⚠️ Windows 用戶：建議用 WSL (Windows Subsystem for Linux)，然後跟 Linux 教學。Native Windows 雖然有但唔穩定。

## 跟住做

### 1. 事前準備 (5 分鐘)

開 Terminal (Mac: iTerm or Terminal.app; Linux: 你 default shell):

```bash
# Check Node.js
node --version
# 需 v18.0.0+. 如果未 install 或者太舊 → install via nvm:
# curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
# nvm install 18

# Check npm
npm --version
# 通常跟 Node install

# Check git
git --version
# 通常 default 會 install。如果冇:
# Mac: xcode-select --install
# Linux: sudo apt install git
```

如果 3 個都 OK → 繼續。如果任何一個 fail → 先解決嗰個。

### 2. Install Claude Code CLI (5 分鐘)

```bash
npm install -g @anthropic-ai/claude-code
```

⚠️ `-g` 即 global install。部分 system 需要 admin permission。

**常見 error**:

❌ `EACCES: permission denied`
✅ Fix: 用 sudo 或者 (最好) 重新配置 npm prefix:
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc  # 或 ~/.bashrc
source ~/.zshrc
```
然後 retry `npm install -g @anthropic-ai/claude-code`

❌ `npm ERR! 404 Not Found`
✅ Fix: 確認 package name 正確 `@anthropic-ai/claude-code` (有 `@anthropic-ai/` prefix)

❌ Install 成功但 `claude` 命令 「not found」
✅ Fix: PATH 未 update。Source 你 shell config:
```bash
source ~/.zshrc  # 或 .bashrc
```
然後 type `claude --version`

### 3. Setup API key (5 分鐘)

去 [console.anthropic.com](https://console.anthropic.com):

1. Sign in
2. Settings → API Keys
3. Create key → label「claude-code-local」
4. **Copy key 即刻儲存好** (只會見到一次)

Set env var:

```bash
# 編輯你 shell config (Mac default zsh):
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.zshrc
source ~/.zshrc

# 確認:
echo $ANTHROPIC_API_KEY
# 應該出你 key
```

⚠️ **Security**: 永遠唔好 commit API key 入 git。加落 `.gitignore`:
```
.env
.env.local
```

### 4. Run 第一個 Claude Code session (10 分鐘)

開一個 test project folder:

```bash
mkdir ~/claude-test
cd ~/claude-test
git init  # optional but recommended
```

Run Claude Code:

```bash
claude
```

第一次 run，Claude Code 會：
- Welcome message
- 你 confirm 工作 directory
- 顯示 default mode (interactive)
- 等你第一個 prompt

試第一個 prompt:

```
整個簡單 Python script print "Hello from Claude Code" 5 次
```

Claude:
- Generate script
- 問你 permission 「create file `hello.py`?」 (Y/N)
- 你答 Y
- File created
- 顯示 file content + path

`ls` 確認 file 確實存在:
```bash
ls
# hello.py
```

Run 試吓:
```bash
python hello.py
# Hello from Claude Code
# Hello from Claude Code
# ...
```

🎉 **你跑咗第一個 Claude Code session**。

### 5. Understand permission model (5 分鐘)

Claude Code 每 read / write / shell command 都會先問 permission:

- **Allow once**: 即此 1 次 OK
- **Allow always**: 呢類 action 後續自動 allow
- **Reject**: 拒絕，Claude 試另一個 approach

新手建議: 開頭全部手動 approve，建立信心。1 個禮拜後你 sense 到個 pattern, 用 「Allow always」 for safe action (read file / git status etc).

⚠️ 永遠唔好 「Allow always」 for `rm`, `git push --force`, `sudo`, 任何破壞性 action。

### 6. Exit + return

`Ctrl+C` exit session。

Folder 入面 state 會保留 (你 create 嘅 file 仲喺度)。下次 `cd` 入同一個 folder + `claude` 繼續。

Claude Code 對 1 個 folder 嘅 conversation history 自動儲存喺 `.claude/` subfolder (記得 gitignore 佢，唔好 commit)。

## 變化

### 變化 1：Connecting Claude Code to your IDE

VS Code integration:

```bash
# Install VS Code extension
code --install-extension anthropic.claude-code
```

開 VS Code，sidebar 出現 Claude Code panel。可以:
- Highlight code → 問 Claude
- Inline AI completion (類似 Copilot 但 model 唔同)

JetBrains (IntelliJ / WebStorm etc) 有類似 plugin。

### 變化 2：Project-specific config (CLAUDE.md)

每個 project 可以有 `CLAUDE.md` file 喺 root，Claude 自動 read：

```markdown
# Project Context

我哋係 Next.js 14 App Router project.

## 主要 conventions
- Server actions 喺 `actions.ts` files
- Tailwind for all styling
- TypeScript strict mode

## Critical files
- `app/layout.tsx` — global layout
- `lib/auth.ts` — auth utilities
- `prisma/schema.prisma` — database schema
```

呢個等如 project-specific system prompt。Claude 每次 session 入呢個 folder 自動 inject context。

### 變化 3：Specific model selection

Default Claude Code 用 latest Claude. 部分 case 你想明確指明 model:

```bash
# 用 specific model
claude --model claude-opus-4-7

# 或在 session 入面:
# /model claude-sonnet-4-6
```

Trade-off:
- Opus = 最強推論, 較慢 + 貴
- Sonnet = 平衡, 大部分 task default
- Haiku = 快, 細 task / 大量 parallel

日常 work: 接受 default。研究 task: 明確用 Opus。
