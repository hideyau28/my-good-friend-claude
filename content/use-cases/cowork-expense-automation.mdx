## 情境

你出咗成個月差，收據散喺手機相簿、email 附件、Apple Notes 度。公司用緊舊式報銷系統，**每月底要交一份 .xlsx**：

- 日期
- 商戶
- 類別（Travel / Food / Office / Client Entertainment / Other）
- 金額（HKD）
- 換算貨幣（如果係 JPY / USD / RMB）
- 項目 / 公幹編號
- 收據附上（Y/N）

正常人花一個鐘整呢份報告，**因為個任務完全係機械式**——開 email、揾收據、對返公幹、抄入 sheet、匯率換算。呢個任務完美貼合 Cowork：**重覆、有規律、唔需要創意**。

呢篇教你設定一個每月底自動行嘅開支報告草稿——**你個最後檢視仍然要做（公司審計需要人手批核），但 80% 嘅機械式工作已經自動完成**。

> 注意：Cowork 仲係研究預覽版。佢會直接讀你 Gmail / 本地檔案。設定之前喺 Mac 開一個專用「expense」資料夾畀佢，唔好畀佢漫遊敏感檔案。

## 跟住做

### 1. 開個專用資料夾 + Apple Notes 區

喺你 Mac Documents 度開：

```
~/Documents/expenses/
├── 2026-03/        ← 呢個月嘅
│   ├── receipts/   ← 你儲相片入嚟
│   └── (Claude 會喺呢度生成草稿)
└── archive/        ← 過往幾個月
```

Apple Notes 開一個叫「Trips & Projects」嘅 note，列出你呢個月有去咗邊：

```
2026-03 Trips & Projects:
- 03/05–03/08 Tokyo (Client X, 項目編號：CX-2026)
- 03/14 Macau 一日遊 (Client Y, 項目編號：CY-Q1)
- 全月 Office (項目編號：OFFICE)
```

呢個 note 係 Claude 嘅參考依據 — 佢用呢個清單去對返收據落公幹。

### 2. 開 Claude Desktop 嘅 Cowork session

入 Cowork 模式（要 Pro / Max plan）。

### 3. 用呢個 prompt 設定流程

<PromptBlock label="完整 prompt — 設定每月 expense 自動化">
我想設定一個每月底自動執行嘅 expense report 任務。以下係流程：

**輸入來源：**
1. 我嘅 Gmail（搵商戶 email、電子收據）
2. ~/Documents/expenses/2026-XX/receipts/ 資料夾（我手動拍照儲落嘅收據相）
3. Apple Notes 入面叫「Trips & Projects」嘅 note（畀你睇我去過邊、編輯過項目編號）

**每月最後一日晚上 9 點自動行：**

第一步：收集收據
- 掃 Gmail 過去 30 日，揾包含「receipt」、「invoice」、「booking confirmation」、「reservation」等字眼嘅 email
- 掃 ~/Documents/expenses/2026-[月份]/receipts/ 資料夾入面所有圖片
- 由每張收據抽出：日期、商戶名、金額、貨幣

第二步：分類
- 用商戶名估類別：
  - 航空公司 / 機票 / 酒店 → Travel
  - 餐廳 / cafe → Food
  - 文具店 / 印刷 / Apple Store → Office
  - 高端餐飲 + 同期有 client meeting → Client Entertainment
  - 其他 → Other（同我確認）

第三步：對返公幹 / 項目
- 用日期對返 Apple Notes「Trips & Projects」
- 將每張收據配落最相關嘅公幹 / 項目編號
- 如果含糊（例如同一日有兩個項目），標「需確認」

第四步：貨幣換算
- 如果唔係 HKD，攞當日嘅匯率（用公開可查嘅匯率）
- 出原幣 + 金額 + 換算 HKD

第五步：輸出
- 喺 ~/Documents/expenses/2026-[月份]/ 度生成「[月份]-expense-draft.xlsx」
- 三個 sheet：
  - **Main**：完整開支清單，欄位同公司範本一致
  - **Flagged**：需要我確認嘅項目（缺收據、含糊類別等）
  - **Summary**：每個類別嘅總額、每個公幹 / 項目嘅總額

第六步：email 提醒
- 寄一封 email 畀我，標題：「[月份] Expense Draft Ready — [N] 條」
- 內容：簡短總結 + flagged 數 + 個檔案路徑

**重要限制：**
- 唔好交任何 expense report 畀公司！草稿出嚟之後等我親自檢視
- 缺收據嘅花費寧願標「未有」，唔好硬作
- 估唔到類別嘅保守標「Other - needs review」
- 貨幣換算用嘅匯率一定要有來源，喺 Notes 欄寫明

設定好嘅話確認我，再排程落每月最後一日 9pm 行。
</PromptBlock>

### 4. 第一次試行

第一次跑唔好排程，要 Claude 喺沙盒模式試行：「即場行一次，展示輸出，唔好真係寫落硬碟」。

睇下：
- 佢有冇正確讀你嘅 Gmail
- 類別估得合唔合理
- 公幹 / 項目對得準唔準
- 輸出 xlsx 格式啱唔啱公司範本

執完 prompt 之後，先啟用排程任務。

### 5. 第一個月最終檢視

第一個月 Claude 生成草稿之後，你親自核對嘅時候要：

1. 開 Flagged sheet — 處理所有含糊項目
2. 開 Main sheet — 隨機抽 5 條核對：商戶名、金額、貨幣對返收據啱唔啱
3. 對返 Summary — 個總數合理性檢查（接近你心目中個月度燒錢？）

通常第一個月會發現 2-3 樣要調整嘅嘢（例：某啲商戶分類錯、某啲公幹你唔記得加入 Notes）。**微調一次 prompt，之後幾個月就順**。

<PromptBlock label="變化 2 — 揾返漏咗嘅收據">

額外步驟：對返我嘅信用卡月結單（已上載做 PDF / 喺 Gmail 嘅月度結單）。

對於結單上嘅扣帳：

### 變化 3：團隊版本（你係主管）

<PromptBlock label="變化 3 — 整個團隊嘅 expense">
（如果你帶緊一隊 5 人團隊，每個都交自己嘅開支）：

每月最後一日：
1. 收齊 5 個下屬個人嘅 expense 草稿（佢哋自己用變化 1 出嘅 xlsx）
2. 合併成一份團隊層面 expense report
3. 計每個項目 / 公幹嘅總共花費
4. 標示：有冇邊個離群（例：某個下屬餐飲花費明顯高過團隊平均）——唔係指控，係幫你準備同財務解釋
5. 出團隊層面總結 email 寄畀我 + 財務伙伴

私隱提醒：呢個任務處理嘅係敏感員工資料，唔好分享其他人嘅個別細節畀錯人。
</PromptBlock>

## 點解呢個任務啱 Cowork 唔啱 Chat

呢個任務嘅特性：
- ✓ 重覆：每月做同一件事
- ✓ 機械式：規則為主、唔需要創意判斷
- ✓ 多來源：要對返 Gmail + 資料夾 + Notes + 日曆
- ✓ 有明確輸出格式
- ✓ 容錯：草稿出錯 OK，你檢視時可以捉返

Chat 模式嘅限制：你要每月主動開 chat、逐張上傳收據、逐個指示佢點做。**做一次唔煩，做 12 次就煩**。

Cowork 嘅好處喺：**設定嗰 30 分鐘嘅投資，換成個年慳返 12 個鐘 + 永遠唔會漏**。

最後一個提醒：Cowork 而家仲係研究預覽版，**唔好交畀佢做關鍵決定**（交開支、發客戶 email、扣錢）。佢最啱嘅就係 **「草擬 + 整理」**，最後一步嘅批核永遠係你。
