## 情境

你個 B2B API 上咗 prod 半年。Endpoint 由 12 條變 47 條。

文件呢？

- `/docs/api.md` —— 最後一次 update 喺 3 個月前
- `openapi.yaml` —— 6 個 endpoint 仲叫舊名
- Frontend 嘅 `api-types.ts` —— 手動 copy 嘅、有兩個 field 同 backend 唔同
- `example-response.json` —— 嗰隻 deprecated `user_id` field 仲喺度

結果：客戶 integration team 第三次寫 email 投訴「why does the response shape not match your docs?」。Frontend 同事每隔幾日撞一次 type mismatch。

問題唔係冇人寫 docs，係**docs 同 code 唔係同一個 source of truth**。每次手動 sync，總有一層漏。

呢篇教用 Claude Code 建立 **3-tier doc sync**：

1. **OpenAPI spec** —— single source of truth，由 route handler 直接生成
2. **TypeScript types** —— spec 出 codegen，frontend / SDK 直接 import
3. **Example payload** —— Claude 跑落 staging、核實個 response，update example

加埋一個 pre-commit hook + `/api-doc` slash command —— 加新 endpoint，docs 自動跟。

## 跟住做

### 1. Route → OpenAPI spec 自動 sync

第一層 source of truth 係 `openapi.yaml`，但唔係手寫，係由 route file 推導。

假設你用 Hono / Express，route 大概咁樣：

```typescript
// src/routes/orders.ts
app.post('/api/orders', async (c) => {
  const body = CreateOrderSchema.parse(await c.req.json())
  const order = await createOrder(body)
  return c.json({ success: true, data: order })
})
```

關鍵：**Zod schema 已經喺度**，OpenAPI 嘅資訊就喺 schema 入面。

加個 `CLAUDE.md` rule 落 project root：

```markdown
## OpenAPI Sync Rule

每次加 / 改 route handler，必須同步更新 openapi.yaml：
1. Route path + method 對應 paths section
2. Zod schema 用 zod-to-openapi 轉換做 requestBody / responseBody
3. 唔好手寫 schema —— 永遠由 Zod source 推
4. 如果 Zod schema 改咗 field，openapi.yaml 對應 section 必須同步
```

Claude Code 讀到呢條 rule，你下次叫佢加 endpoint 嘅時候，佢會**自動更新 spec**。

裝 zod-to-openapi：

```bash
npm install @asteasolutions/zod-to-openapi
```

呢個 lib 等 Zod schema 直接 emit OpenAPI fragment，Claude 唔需要重複寫 schema 兩次。

### 2. TypeScript types codegen

OpenAPI 有咗，frontend types 直接由佢生。

```bash
npm install -D openapi-typescript
```

加條 script 落 `package.json`：

```json
{
  "scripts": {
    "gen:types": "openapi-typescript openapi.yaml -o src/types/api.ts"
  }
}
```

跑：

```bash
pnpm gen:types
```

`src/types/api.ts` 入面會出晒所有 endpoint 嘅 request / response 型別：

```typescript
export interface paths {
  '/api/orders': {
    post: {
      requestBody: { content: { 'application/json': components['schemas']['CreateOrder'] } }
      responses: { 200: { content: { 'application/json': components['schemas']['Order'] } } }
    }
  }
}
```

Frontend / SDK 直接 import：

```typescript
import type { paths } from '@/types/api'
type CreateOrderBody = paths['/api/orders']['post']['requestBody']['content']['application/json']
```

**手動 copy field 嘅日子完。** Spec 改一行，types 即時跟。

### 3. Example payload test + 核實 pipeline

最後一層：example response。呢層最易腐，因為冇人想手跑 endpoint 再 copy JSON。

寫條 script `scripts/refresh-examples.ts`：

```typescript
import { spec } from '../openapi'

for (const endpoint of spec.paths) {
  const response = await fetch(`https://staging.api.example.com${endpoint.path}`, {
    method: endpoint.method,
    headers: { Authorization: `Bearer ${process.env.STAGING_TOKEN}` },
    body: endpoint.exampleRequest,
  })
  const data = await response.json()
  await writeFile(`docs/examples/${endpoint.id}.json`, JSON.stringify(data, null, 2))
}
```

跑：

```bash
pnpm tsx scripts/refresh-examples.ts
```

跟住叫 Claude Code 核實：

<PromptBlock label="Verify example payload prompt">
讀 docs/examples/ 入面所有 JSON 同 openapi.yaml 嘅 response schema。

對比每一個 example：
- 有冇缺 field（spec 講有、example 冇）
- 有冇多 field（example 有、spec 冇）—— 通常代表 spec 落後
- Field type 有冇對唔上

逐個 endpoint 出 diff report，列明邊個要改邊一層。
</PromptBlock>

Claude 會逐個檢、出 markdown report、指明「呢個 example deprecated field 要移除」或「spec 漏咗 `created_at`」。

呢個三層循環跑完一次，docs 同 code 完全對齊。

### 4. /api-doc slash command 加新 endpoint 一句話搞掂

最後加個 project-level slash command。喺 `.claude/commands/api-doc.md`：

```markdown
---
description: 加新 API endpoint + 自動 sync 三層 docs
---

加新 endpoint 嘅 workflow：

1. 喺 src/routes/ 加 route handler（Zod schema 必寫）
2. Run pnpm openapi:emit —— openapi.yaml 自動更新
3. Run pnpm gen:types —— TypeScript types 自動 codegen
4. Run pnpm tsx scripts/refresh-examples.ts —— example payload 跑落 staging
5. Verify：對比 example vs spec，出 diff
6. Commit：openapi.yaml / types / examples 三層同 route handler 一齊入 PR

用家會用一句話 describe endpoint。你完成 1-6 步先返人。
```

之後 Claude Code 入面：

```
/api-doc 加一個 GET /api/orders/:id endpoint，攞返單筆訂單詳情，permission 同 POST 一樣
```

Claude 會跑晒成個 chain：route → schema → spec → types → example → 核實 → commit。

收工。

加埋一個 pre-commit hook（`.husky/pre-commit`）：

```bash
#!/bin/sh
pnpm openapi:emit
pnpm gen:types
git diff --exit-code openapi.yaml src/types/api.ts || {
  echo "❌ Docs out of sync. Commit auto-updated files."
  exit 1
}
```

任何 PR 入 main，docs 唔同步直接 block。

## 變化

### 變化 1：GraphQL schema 版（vs REST）

GraphQL 唔需要 OpenAPI —— `schema.graphql` 本身就係 source of truth。但**同步 example query / response** 嘅問題一樣存在。

Claude Code workflow：

```
讀 schema.graphql，對每個 Query / Mutation 生成：
1. Example query (with fragments)
2. 落 staging 跑、record response
3. 寫入 docs/graphql-examples/
```

加 codegen（`@graphql-codegen/cli`）出 TypeScript types，pattern 同 REST 版完全一樣。

### 變化 2：Python FastAPI 版（Pydantic 同 schema）

FastAPI 識自己 emit OpenAPI —— `/openapi.json` 即係 source of truth。

```bash
curl http://localhost:8000/openapi.json > openapi.json
```

之後 chain 同 TypeScript 版一樣：openapi-typescript-codegen 出 client types，Claude 核實 example。

關鍵：Pydantic model 同 endpoint signature 已經夠表達，唔需要再加 layer。Claude 直接讀 router file，spec 同步喺 import 嘅 Pydantic schema。

### 變化 3：Webhook docs（asymmetric send/receive）

Webhook 同 REST 唔同 —— **你係 send 出去**，但收嘅人要知你 payload shape。

呢類 docs 要分兩面：

- **outgoing** —— 你發乜（payload schema + 例子）
- **expected response** —— 對方應該回乜（success / retry）

用 AsyncAPI spec（係 OpenAPI 嘅 event 版本）。Claude Code workflow：

```
讀 src/webhooks/ 嘅 publisher code，提取每個 event 嘅：
- Topic / event name
- Payload Zod schema
- Retry policy
出 asyncapi.yaml，跟住生 markdown docs 畀 partner team
```

Partner integration 接 webhook，永遠 reference 自動同步嘅 docs。

## 一個心態

API docs 過時嘅根源唔係懶 —— 係 docs 同 code **本身就喺兩個 source**。手動 sync 必有漏。

呢個 workflow 嘅 insight：**將 single source 嘅範圍縮到一層**（route handler 入面嘅 Zod schema），其他層全部係 derived artifacts。Derived 嘅嘢 Claude 識生，唔需要人工維護。

- ✅ **Schema-first**：Zod / Pydantic 一定要喺 route handler 入面定義。手寫 JSON schema 永遠落後。
- ✅ **Codegen everything**：types / examples / markdown docs 全部由 spec 生。
- ✅ **Pre-commit block**：docs 唔同步嘅 PR 唔可以入 main。CI 都要 enforce。
- ⚠️ **Staging endpoint 要 stable**：example refresh 跑落真實 staging，唔可以連 dev。Token 用 read-only。

下個 sprint 加新 endpoint 嘅時候，試 `/api-doc` 一句話搞掂。半年之後再望，47 條 endpoint 嘅 docs 仲係同 code 對齊。客戶 integration team 唔再寫投訴 email。
