## 情境

你係 SaaS 或者電商創辦人，跟住 5 個 direct competitor：

- 佢哋幾時加價、減價
- 邊個出咗新 pricing tier
- Landing page 嘅 hero copy 點變（反映佢哋市場 positioning 點轉）
- 邊個出咗新 feature、邊個悄悄 sunset 咗一個 plan

呢類資訊唔係競爭機密，係佢哋自己擺喺公開網頁。問題係你要每週親手開 5 個 tab、scroll、憑記憶對比上週見到嘅版本——通常做兩個月就會放棄。

呢個 chore 嘅特性啱晒 Cowork：重覆、結構化、輸出明確（一頁 brief）、容錯率高（出草稿你睇）。每週一朝 7am 跑一次，你 9am 返公司就有杯咖啡同一頁 competitive brief 等你。

> 利益申報：呢篇講嘅係讀**公開** landing page + pricing page。唔好用呢個流程 scrape 需要登入嘅 dashboard、付費內容、或者你冇 access 嘅 staging URL。技術上 Claude 唔會幫你做呢啲，但都要講清楚：呢條線過唔得。

## 跟住做

### 1. 整理 competitor list + snapshot baseline

第一步係手動嘅，做一次就得。開一個 markdown file：

```
~/Documents/competitive-monitor/
├── competitors.md         ← 你要 track 嘅 URL list
├── snapshots/
│   ├── 2026-W21/          ← 每週嘅 fetch 結果
│   └── archive/           ← 舊週嘅
└── briefs/                ← Cowork 出嘅週報
```

`competitors.md` 應該係：

```
## Competitor A
- Landing: https://competitor-a.com
- Pricing: https://competitor-a.com/pricing
- Changelog: https://competitor-a.com/changelog
- Positioning: 「all-in-one for SMB」
- 上次 major change: 2026-04 加咗 enterprise tier

## Competitor B
...
```

「Positioning」同「上次 major change」呢兩個欄好重要——畀 Cowork 一個基線，知道乜叫「值得 flag」、乜叫「行貨小改」。

### 2. 核心 diff prompt

<PromptBlock label="完整 prompt — 每週 competitor monitor">
我要設定一個每週一朝 7am 自動跑嘅 competitor monitoring 任務。

**輸入：**
- `~/Documents/competitive-monitor/competitors.md`（5 個競爭對手嘅 URL + 我嘅背景 notes）
- `~/Documents/competitive-monitor/snapshots/` 入面上週嘅 fetch 結果（用嚟對比）

**每週一 7am 跑：**

第一步：fetch 每個 competitor 嘅 landing page + pricing page + changelog（如果有）
- 只讀 public URL
- 抽出純文字內容（去走 nav / footer / cookie banner 等 noise）
- 存落 `snapshots/2026-WXX/competitor-a-landing.txt` 等

第二步：對比上週版本
- 同 `snapshots/2026-W(XX-1)/` 入面對應 file 做 diff
- 識別三類變化：
  1. **Pricing change** — 任何數字、tier name、feature gating 嘅改動
  2. **Positioning change** — hero headline、subheadline、main CTA 嘅改動
  3. **Feature announcement** — 新 feature copy、新 page、changelog 入面嘅 entry

第三步：每個變化寫 3-5 句 brief
格式：
- **[Competitor] — [變化類別]**
- 變化內容（before → after）
- 我嘅 take（呢個對我哋 positioning 嘅意義，1-2 句）
- 建議 follow-up（要唔要應對 / 觀察就得 / 唔關事）

第四步：出一頁 weekly brief
- File path：`briefs/2026-WXX-competitive-brief.md`
- 最頂 TL;DR：3 句講晒呢週最值得知嘅
- 跟住每個 competitor 一個 section
- 最後 section：「冇變化」嘅 competitor 都要列（confirm 我冇 miss）

第五步：email 我
- Subject：「Week XX competitive brief — [N] changes detected」
- Body：上面 TL;DR + brief 嘅 file path
- 唔好 attach 成個 brief，等我自己開 file 睇

**重要限制：**
- 只 fetch 我 list 入面嘅 URL，唔好自己 follow link 去其他頁
- 唔好嘗試登入任何 gated content
- 如果某個 URL 收 403 / 404，flag 出嚟唔好猜內容
- Pricing 嘅變化要引用原文（quote 數字 + tier name），唔好 paraphrase
</PromptBlock>

### 3. Weekly brief format — 你 9am 睇 5 分鐘

第一次跑完之後檢查 output 嘅排版。理想嘅 brief 應該係：

```
# Week 21 Competitive Brief (2026-05-25 → 2026-05-31)

## TL;DR
- Competitor A 加咗 $20/月喺 Pro tier，feature 無增加
- Competitor C 改咗 hero copy，由「for teams」轉「for enterprises」
- 其餘 3 個 competitor 無變化

## Competitor A — Pricing change
Pro tier：$49 → $69/月。Feature list 完全冇變。
我嘅 take：純粹加價測試，可能反映佢哋 retention 唔錯所以敢試。
建議：觀察兩週睇有冇回滾。我哋 Pro tier 暫時唔需要跟。

## Competitor C — Positioning change
Hero：「Project management for teams」→「Enterprise-grade project management」
Sub：CTA 由「Start free」變「Talk to sales」
我嘅 take：明顯上移 ICP，可能放棄 SMB 市場。
建議：呢個係我哋 SMB 嘅機會窗口，下季 marketing 可以揀呢個角度打。

...
```

3-5 句一個 entry。多咗你唔會睇，少咗冇 context。

### 4. Monthly strategic review — 用 4 週 brief 做 trend 分析

每週嘅 brief 各自獨立冇咩用——真正嘅 value 喺月底嘅 trend。

每個月最後一個禮拜五跑多一個 task：

<PromptBlock label="月度 trend 整合">
讀取 `~/Documents/competitive-monitor/briefs/` 入面過去 4 週嘅 brief。

整一份 monthly strategic memo：

1. **市場趨勢**：5 個 competitor 加埋睇，呢個月有冇 collective movement？（例：3 個都加價 → 行業 willingness to pay 上升）
2. **個別 competitor 嘅戰略軌跡**：每個 competitor 過去 4 週嘅變化串起嚟，反映佢哋 roadmap 嘅方向
3. **空白市場**：5 個 competitor 都冇做嘅 segment / feature / positioning，可能係我哋嘅機會
4. **威脅信號**：邊個 competitor 嘅變化最值得我哋警戒，點解

寫畀我自己睇——直接、有立場、唔好「呢個視乎」嘅模稜兩可。
</PromptBlock>

呢一份 memo 一個月做一次，係你季度規劃會議嘅重要 input。

## 變化例子

### 變化 1：加埋廣告監察

<PromptBlock label="變化 1 — 加埋 Meta ads library + Google ads transparency">
喺原本流程上面加多一步：

第一步加：fetch Meta Ad Library 入面每個 competitor 嘅 active ads（公開資料）
- 用 URL：https://www.facebook.com/ads/library/?q=[competitor name]
- 抽出 ad creative 嘅 text、started running 日期
- 留意：同上週對比，邊啲 ad 新出、邊啲落咗、邊啲改咗 copy

第二步加：Google Ads Transparency Center 做同樣嘢
- URL：https://adstransparency.google.com/?domain=[competitor.com]

加入 brief 嘅新 section：「廣告動向」
- Active ads 嘅 copy 主題（reflect 緊乜 messaging）
- 新出嘅 ad 反映乜 campaign focus
- 落咗嘅 ad 反映乜 messaging 唔 work

呢個對 paid acquisition 嘅 startup 特別有用——你睇到 competitor 嘅 paid messaging 變化，往往比 landing page 更快反映佢哋 positioning 嘅 test。
</PromptBlock>

### 變化 2：加埋新聞 + 社交 mention

<PromptBlock label="變化 2 — 加新聞 + 社交監察">
我想擴展呢個 monitor 去包：

- Google News：每個 competitor 名搜過去 7 日（用 site:news.google.com 嘅 query format）
- Reddit：subreddit 入面有冇 mention competitor（例：r/SaaS、r/Entrepreneur、行業特定 sub）
- LinkedIn：competitor 公司 page 嘅新 post（公開內容）

加入 brief 嘅新 section：「外部聲音」
- 媒體覆蓋（PR placement、funding 公告、人事變動）
- 用戶討論：邊啲係 positive、邊啲係 negative、有冇 recurring complaint
- 我嘅 take：呢啲 external signal 同 landing page 變化加埋睇，反映 narrative 嘅 alignment

注意：唔好嘗試讀 private LinkedIn post、唔好爬需要登入嘅 forum。
</PromptBlock>

### 變化 3：自家 product team 內部用

<PromptBlock label="變化 3 — 跟自己嘅 release notes">
我哋自己係 SaaS，每個 sprint 出 release notes。

請改造呢個流程嚟做內部用：
- 監察我哋自己嘅 changelog page + 過去 4 週嘅 release notes
- 同我哋自己嘅 positioning doc（內部 PRD / messaging guide）對比
- Flag：邊啲我哋出嘅 feature 喺 marketing site 上面冇反映、邊啲 messaging 過時、邊啲 pricing tier 嘅 feature list 同實際 product 唔 match

每週一畀我哋 product / marketing alignment brief：
- 「呢個 feature 上週出咗，但 marketing page 入面冇提」
- 「Pricing page 寫嘅 Pro tier 包 X，但 X 已經 sunset 咗」
- 「我哋自己嘅 hero copy 同我哋上月嘅 positioning workshop 結論已經 drift」

呢個對 10-30 人嘅 startup 特別有用——marketing / product / sales 嘅 alignment 通常喺呢個 size 開始 drift，定期 audit 比起每季打仗高效得多。
</PromptBlock>

## 點解競爭監察啱 Cowork 做

> 競爭監察呢樣嘢嘅本質係：低 stakes 嘅單一動作（fetch 一個 URL、做一次 diff），但要長時間累積先有用。Cowork 嘅優勢就喺度——人類做兩個月會放棄，自動化做兩年都唔會攰。但 5 個 competitor 嘅月度 strategic memo 你一定要自己讀、自己諗。AI 出 trend，你出判斷，呢個分工先 work。

最後一個提醒：呢個流程嘅 ethical 線好清楚——讀公開頁面 OK、爬登入後內容唔 OK。Cowork 唔會自動越過呢條線，但你 prompt 入面唔好試引佢去。市場研究嘅長期可持續性，靠你守住呢條線。
