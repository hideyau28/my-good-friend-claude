## 情境

你係：
- **律師 / 法律助理**：客戶文件喺法律上唔可以上傳落雲端
- **醫療 / 診所行政**：病人紀錄受《個人資料（私隱）條例》（PDPO）規管
- **金融 / 合規審查**：內部模型 / 風險文件唔可以離開公司網絡
- **政府 / 非政府組織**：受特定數據治理限制

但你想用 AI 工作流程：做摘要、翻譯、修飾文字。雲端嘅 Claude / ChatGPT 唔可以用。

呢篇教你設定**本地大型語言模型 (Local LLM) 喺 Mac 運行**——開源模型（Llama / Mistral）跑喺你部機度，零雲端傳輸。

> ⚠️ 注意：本地模型嘅質素同最新嘅 Claude / GPT-4 有差距（特別係推理論證同多語言能力）。但處理日常任務已經夠用。設定之前要調整期望：呢個係「私隱第一，質素第二」。

## 跟住做

### 1. 硬件檢查

最低要求：
- Mac M1 / M2 / M3（Intel 晶片嘅 Mac 跑得超慢，唔推薦）
- 最少 8GB 記憶體 (RAM)，16GB 跑 7B 模型 OK，32GB 以上跑 70B 模型
- 30GB 空餘硬碟空間（每個模型大約 4-8GB）

如果你部 Mac 唔符合要求，呢個設定唔啱你。請留在雲端並加簽企業數據協議。

### 2. 安裝 Ollama

[Ollama](https://ollama.com) 係 macOS 嘅本地模型運行工具，係最簡單嘅設定方式。

```bash
# 透過 Homebrew 安裝
brew install ollama

# 啟動服務
ollama serve

# 開第二個 terminal：下載 Llama 3.1 8B
ollama pull llama3.1:8b

# 測試
ollama run llama3.1:8b
> 你好
```

大約 5-10 分鐘設定時間。

### 3. 模型對比與選擇

```bash
# 中文友善
ollama pull qwen2.5:7b    # 阿里嘅 Qwen，中文強
ollama pull yi:6b         # 推理能力強

# 通用
ollama pull llama3.1:8b
ollama pull mistral:7b

# 大模型（需要 32GB 以上 RAM）
ollama pull llama3.1:70b
```

每個模型我嘅實際評測（主觀）：
- **Qwen 2.5 7B**：中文最強，廣東話書面語算通順。做文件摘要好啱用
- **Llama 3.1 8B**：英文最強，寫 code 唔錯。中文還可以，但未夠地道
- **Mistral 7B**：兩者之間，容量最細

### 4. 用呢個提示詞設定工作流程

<PromptBlock label="完整 prompt — 設定本地 LLM 工作流程">
我裝咗 Ollama，拉咗以下模型：[列出]。我嘅用途：

[填寫具體用途：例如「摘要法律文件」、「修飾醫療病人筆記」、「翻譯內部合規文件」]

請：
1. 推薦邊個模型最適合我嘅用途（基於容量 / 語言 / 推理能力）
2. 整一個簡單嘅 terminal 工作流程：
   - 我有一份 .txt 或 .md 文件
   - 我想將檔案內容傳入模型，輸出摘要 / 修飾 / 翻譯
   - 唔需要整介面，純命令列 (command line) 就可以
3. 寫一個 bash 函式去簡化呢個工作流程（加入到 `~/.zshrc`）
4. 注意位：本地模型嘅典型限制（上文下理長度、冷門主題嘅準確度）
5. 幾時應該轉返用雲端：邊種任務本地模型明顯唔夠強，需要雲端（我會同公司合規團隊確認）

我嘅 Mac 配置：[M1 / M2 / M3，記憶體大小]
</PromptBlock>

### 5. 建立簡單嘅 bash 函式

```bash
# 加入到 ~/.zshrc
function ask-local() {
  local prompt="$1"
  local file="$2"
  
  if [ -z "$file" ]; then
    echo "$prompt" | ollama run qwen2.5:7b
  else
    cat "$file" | ollama run qwen2.5:7b "$prompt"
  fi
}

# 使用方法：
ask-local "用廣東話書面語做 3 句摘要：" my-doc.txt
ask-local "Translate to English keeping legal precision:" contract.txt
```

呢個係最精簡嘅工作流程，5 分鐘搞掂。

## 一個取捨框架

本地模型唔等於雲端 Claude 嘅替代品。兩者根本上係唔同嘅取捨：

| 維度 | 雲端 Claude | 本地 Llama |
| --- | --- | --- |
| 質素 | ★★★★★ | ★★★ |
| 速度 | ★★★★★ | ★★ |
| 私隱 | ★ | ★★★★★ |
| 成本 | $20-50/月 | 一次性硬件投入 |
| 設定阻力 | 低 | 中 |

決策原則：**任何「如果洩漏，會有嚴重後果」嘅任務 → 用本地**。日常任務 → 用雲端。

呢個劃分唔需要非黑即白。90% 嘅香港打工仔工作，用雲端完全無問題。5% 敏感個案用本地。

最後嘅核心思維：本地模型嘅魔力唔係取代 Claude，係解鎖「之前因為私隱問題而禁止使用 AI」嘅領域。對香港法律 / 醫療 / 金融行業嚟講，呢個解鎖實實在在。
