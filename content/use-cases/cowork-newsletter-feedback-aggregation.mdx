## 情境

你一個人寫 newsletter 已經第三個月。每星期三朝早 send 出去之後，反饋散落三個地方：

- **Reply**：30 封左右，由「呢期勁啱睇」到「呢個 framework 我用咗一個禮拜，分享我嘅情況」都有
- **Unsubscribe**：5 個。平台有時會問原因、有時冇
- **New subscriber**：12 個。來源散落 referral link、Twitter bio、朋友轉發

你心知肚明：呢三條資料流加埋先係讀者真正講緊乜。但要一條條 reply 揭、一個個退訂原因抄、再對照邊度嚟嘅 new sub——一個鐘冇得走。Send 完 newsletter 已經攰，星期三再加一個鐘做 review，做兩個禮拜就會放棄。

呢一個鐘係**重覆嘢**：同樣三個來源、同樣歸類邏輯、淨係內容唔同。**Cowork 嘅賣點就係呢度：設定一次，每週日 6pm 自動跑、整好一頁 weekly review markdown、放入你個閱讀 folder**。

> 提提你：Cowork 仲係 research preview，要 Pro/Max plan。佢有 agentic 能力會直接讀你嘅 email 同平台 exports，**開一個專用工作 folder 畀佢**，唔好畀佢掂你其他敏感檔案。

## 跟住做

### 1. 連 newsletter 平台 + Cowork 設定

主流平台（Beehiiv / Substack / ConvertKit / Buttondown）都有兩條路：

- **CSV export**：每週日手動下載一次 subscribers + unsubscribes report，放入專用 folder（例如 `~/Newsletter/Weekly/`）
- **Email forwarding**：將「new subscriber」、「unsubscribe」、「reply」嘅通知 email 轉寄去一個專用 Gmail label（例如 `newsletter-feedback`）

轉寄法慳工夫但要等一個禮拜砌齊。CSV 法即刻有舊資料補返，但要記得下載。我建議**頭兩個禮拜用 CSV，第三個禮拜起改轉寄**。

Cowork 嗰邊：開 Claude Desktop → 揀「Cowork」模式 → 指定工作 folder 就係上面個 `~/Newsletter/Weekly/`。佢淨係睇得到呢個 folder 同你開咗權限嘅 Gmail label。

### 2. 核心 prompt — 歸類 reply、歸類退訂原因

一次過畀佢完整指示。寫清楚三條資料流各自做乜。

<PromptBlock label="完整 prompt — Weekly feedback aggregation">
我 solo 寫一個 HK newsletter（每星期三 send），主題：[填]，訂閱人數：[填]。

我想你每個星期日下午 6 點自動做以下嘅嘢：

**Stream 1：Reply 內容**
睇 Gmail label「newsletter-feedback」入面過去 7 日所有「Re:」開頭嘅 email。每封：

- 抽出 reader email、引用嘅段落（如果有）、佢自己寫嘅內容
- 情緒：正面 / 中性 / 帶批評 / 純分享自己經歷
- 主題歸類：將 30 封 reply 歸入 3-5 個主題（例如「呢期 framework 用得着」、「想睇延伸內容」、「分享自己經歷」、「質疑某個論點」）
- 每個主題出 2-3 條代表性引文（保留原文，唔好改寫）

**Stream 2：Unsubscribe**
睇平台轉寄過嚟嘅 unsubscribe 通知 + folder 入面嗰個 CSV：

- 如果有原因欄位：歸類做（內容唔啱、太頻密、太疏、寫得太長、其他）
- 如果冇原因：標「未填」
- 交叉對照：呢啲退訂係咩時間訂閱？訂閱咗幾耐？（睇得到就標出嚟）
- 警號：如果某一個原因連續兩週上升，紅字標出

**Stream 3：New subscriber**
睇平台嘅 new subscriber list：

- 來源分佈（referral link、direct、social、unknown）
- 如果來源係 referral：邊個現有 subscriber 帶嚟最多
- 同 reply stream 交叉比對：有冇 new sub 係由某條被引文嘅讀者帶嚟（佢哋有冇轉發）

**輸出**
出一份 markdown 報告，存喺呢個 folder，檔名「[YYYY-MM-DD]-weekly-review.md」。格式：

1. 一行 TL;DR（呢個禮拜最值得留意嘅一件事）
2. Reply 主題 + 引文
3. Unsubscribe 規律 + 警號
4. New sub 來源 + 增長 vs 上週
5. 「下週 content 建議」3 條 — 基於 reply 主題同退訂原因推導

注意：唔好自動回覆任何讀者、唔好 send 任何 email、唔好改平台設定。淨係讀取 + 寫 markdown 報告。

設定好同我核對，預演一次先排去每星期日 18:00 自動跑。
</PromptBlock>

第一次跑**一定要預演**。睇下主題歸類嘅準確度、引文有冇剪斷、來源認得啱唔啱。OK 先啟用排程任務。

### 3. Weekly markdown 報告格式 + 交付

預期 output 大致係：

```
# 2026-05-31 · Weekly Review

## TL;DR
呢個禮拜 reply 入面有 6 封提到想睇「framework 點 apply 落非 tech 行業」——
連續第二週呢個 cluster 上升，下週可以做。

## Reply clusters（30 封）
### Framework 應用（12 封）...
### 想睇延伸內容（8 封）...
### 質疑「X 論點」（4 封）...

## Unsubscribe（5 個）
- 太頻密：2
- 內容唔啱：1
- 未填：2
警號：冇

## New subscriber（12 個）
- Referral: 7（其中 4 個由 @reader_handle 帶嚟）
- Direct: 3
- Social: 2

## 下週 content 建議
1. ...
2. ...
3. ...
```

Markdown 好處：你開 Obsidian / Bear / Apple Notes 都讀得到，唔使再開 dashboard。每週累積成一份回顧記錄，月尾做季度回顧直接睇返。

### 4. 回覆策略 — 用洞察去諗下週 content

報告出咗，下一步唔係即刻回覆 30 個讀者，係**分流**：

- **「呢期勁啱睇」純鼓勵**：一次過講聲多謝，30 秒搞掂
- **分享自己經歷嘅**：呢啲最寶貴。逐封回覆，問深入啲，攞授權引去下期 newsletter
- **質疑論點嘅**：揀 1-2 封認真回，可能就係下期題目嘅種子
- **想睇延伸**：記入下週 calendar，唔即刻回

報告入面「下週 content 建議」3 條，**唔好照單全收**。Claude 由規律推出嚟，但你了解自己嘅風格。揀 1 條同你 editorial calendar 啱嘅；剩兩條當後備清單。

## 變化

### 變化 1：加 Twitter / Threads / IG comments

<PromptBlock label="變化 1 — 跨平台反饋">
（同上面 prompt 一樣）再加 Stream 4：跨平台提及

睇我嘅社交帳號（Twitter / Threads / IG）過去 7 日：
- 任何引用、quote tweet、reply、IG comment 提到呢期 newsletter
- 抽出平台、handle、引用片段、情緒
- 歸入返 Reply stream 嘅同一套主題分類（reply 同社交加埋睇走勢）

注意：唔好回覆任何社交帖文，淨係抽資料。
</PromptBlock>

社交平台嘅觸及面有時比 reply 多。加埋一齊歸類，先睇到完整全貌。

### 變化 2：付費 subscriber 版本（focus on 留存）

<PromptBlock label="變化 2 — 付費留存訊號">
（同前面 prompt，但 Stream 2 / 3 改成）：

**Stream 2：退訂取消**
- 區分免費退訂 vs 付費取消
- 付費取消：訂閱咗幾耐先取消？（流失時機好關鍵）
- 警號：如果付費取消連續兩週 > 新增付費訂閱，紅字
- 交叉對照：取消嗰啲付費讀者嘅回覆紀錄（佢哋之前有冇提過憂慮）

**Stream 3：新付費 + 升級**
- 邊個免費讀者升級咗去付費（呢個係領先指標）
- 升級之前一週讀過邊期（揾出「轉化文章」）
</PromptBlock>

付費 newsletter 嘅指標完全唔同。留存大過獲客，流失時機大過淨計退訂數。

### 變化 3：課程 / 會員制 creator 版

<PromptBlock label="變化 3 — 課程反饋匯整">
（同前面 prompt，但三條 stream 改成）：

**Stream 1：課堂完成度**
- 過去 7 日邊堂完成率最高 / 最低
- 流失規律：學員停喺邊一個 module
- 交叉對照：流失學員嘅報名日期（新 vs 舊）

**Stream 2：提問 / 討論**
- 課程平台入面學員問題 / community 帖
- 歸類主題：概念混淆 / 想要更多例子 / 申請延期 / 技術問題
- 揾出「重覆問題」（3 個以上學員問同一條）—下週直接出 FAQ

**Stream 3：新報名 + 退款**
- 新報名來源
- 退款要求（如果有）+ 原因
</PromptBlock>

課程嘅反饋循環仲緊要，因為學員付咗錢期望高。重覆問題直接變下一堂 content。

> Newsletter / 課程嘅長期遊戲：**唔係寫得多快，係聽得多細**。30 封 reply、5 個 unsub、12 個 new sub 加埋，每週都喺度同你講同一件事——但要你停低睇先聽到。Cowork 做嗰一個鐘嘅聆聽工夫，你做嗰 10 分鐘嘅編輯判斷。
