## 情境

你 Claude Code 用咗 1 個月。流程：

- 第 1 日：「Claude review 我嘅 code」✓
- 第 7 日：「Claude 寫 commit message」✓
- 第 14 日：「Claude refactor 呢個 function」✓
- 第 21 日：「Claude check 我哋 Notion task 板」❌

Claude 自己唔識讀 Notion。你要手動 copy task → 貼 → 問 Claude。慢。

呢個限制嘅根源：**Claude 嘅 default 動作只限於 read / write 檔案 + 跑 shell + web search**。外部 SaaS 服務（Notion / Slack / GitHub API / database / etc）要特定整合層。

呢個層就係 **MCP（Model Context Protocol）** —— Anthropic 推嘅開放標準，等 Claude（同其他 AI）用標準方式連外部工具。

**概念**：
- **MCP server** = 一個程序，向 AI 提供特定工具（例如「讀 Notion 任務」、「建立 GitHub issue」）
- **Claude Code** = MCP server 嘅使用方
- **你** = install MCP server + 喺 Claude 註冊

呢篇拆 5 個最有用嘅 MCP server + 10 分鐘 install。

## 跟住做

### 1. MCP 架構速覽

```
[Claude Code] ←── MCP ──→ [MCP server (Notion)] ←── API ──→ [Notion]
[Claude Code] ←── MCP ──→ [MCP server (Slack)]  ←── API ──→ [Slack]
[Claude Code] ←── MCP ──→ [MCP server (Postgres)] ←── SQL ──→ [DB]
```

Claude Code 一個用家，可以同時連多個 MCP server。每個 server 提供各自嘅工具集。

### 2. 頭 5 個最有用嘅 MCP

**1. GitHub MCP** —— repo 瀏覽、issue / PR 管理、程式碼搜索

用途：
- 「列出我哋 repo 嘅未處理 PR」
- 「讀團隊另一個 project 嘅 README」
- 「建立 issue 標題 X、內文 Y」

**2. Notion MCP** —— 任務 / 文檔 / database 讀寫

用途：
- 「show 我哋團隊嘅任務板」
- 「將任務 X 標記做完成」
- 「搜尋文檔關於『API key handling』」

**3. Slack MCP** —— channel 瀏覽、發訊息、搜歷史

用途：
- 「搜 #engineering 關於 Redis 嘅討論」
- 「將每日 standup 發落 #team」
- 「讀上個鐘嘅 DM」

**4. Filesystem MCP** —— 跨資料夾操作（超出 Claude Code 預設）

用途：
- 「比較 /project-a vs /project-b 嘅結構」
- 「同步 5 個姐妹 project 嘅 README」

**5. Postgres / SQLite MCP** —— 直接 query 資料庫（有唯讀安全選項）

用途：
- 「Query staging DB，揾上星期建立嘅用家」
- 「show `orders` 表嘅結構」
- 「跑健康檢查 query」

（其他值得一提：Filesystem、Brave Search、Sentry、Linear、Stripe —— 社區 + 官方混合。）

### 3. Install 你第一個 MCP —— GitHub MCP

GitHub MCP 通常最普遍有用。安裝：

```bash
# 大部分 MCP servers npm 或 pip
npm install -g @modelcontextprotocol/server-github
```

設定 GitHub token：去 [github.com/settings/tokens](https://github.com/settings/tokens)，建立 personal access token，scope 揀 `repo`。儲好 token。

### 4. 喺 Claude Code 註冊 MCP

打開 Claude Code 設定：

```bash
claude --config
# 或手動 edit ~/.claude/settings.json
```

加 MCP server entry：

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "ghp_yourtokenhere"
      }
    }
  }
}
```

存好 + 重新啟動 Claude Code：

```bash
claude
```

Claude Code 載入時會自動偵測 MCP。第一個 prompt 試：

```
List open PR 喺 hideyau28/my-good-friend-claude repo
```

如設定 OK，Claude 透過 GitHub MCP 查 GitHub API → 出 PR 列表。

### 5. 驗證 MCP 行緊

喺 Claude Code 對話入面打：

```
/mcp
```

顯示已連線嘅 MCP 同佢哋嘅工具。應該見到「github」喺度，附帶 `list_pull_requests`、`create_issue` 等工具。

如唔顯示：睇錯誤紀錄
```bash
claude --verbose
```

### 6. 權限管理

每個 MCP 工具第一次叫起，Claude Code 會問權限：

- `Allow once`
- `Allow always for this MCP server`
- `Reject`

新手建議：唯讀操作（list / search / read）→「Allow always」安全。任何改動（create / update / delete）→ 每次手動 approve。

### 7. 試第二個 MCP —— Notion

安裝 Notion MCP：

```bash
npm install -g @modelcontextprotocol/server-notion
```

攞 Notion integration token：[notion.so/my-integrations](https://www.notion.so/my-integrations) → Create new integration → copy「Internal Integration Secret」

加落 Claude Code 設定：

```json
{
  "mcpServers": {
    "github": { ... },
    "notion": {
      "command": "mcp-server-notion",
      "env": {
        "NOTION_API_KEY": "secret_yourtokenhere"
      }
    }
  }
}
```

⚠️ 同樣，每個 Notion 頁 / database 要喺 Notion 介面明確「share with integration」—— 用家經常唔記得呢一步。冇 share = 冇權限。

重啟 Claude Code，測試：
```
List task 喺我「Engineering 2026」Notion database
```

## 變化

### 變化 1：由 GitHub 安裝社區 MCP

唔限於 npm 發佈嘅 MCP。社區開發嘅 MCP 多數喺 GitHub：

```bash
# Clone community MCP repo
git clone https://github.com/user/their-mcp-server
cd their-mcp-server
npm install
```

用絕對路徑註冊：

```json
{
  "mcpServers": {
    "custom": {
      "command": "/full/path/to/their-mcp-server/dist/index.js"
    }
  }
}
```

### 變化 2：Build 自己嘅 MCP server

Anthropic 提供 MCP SDK（Python + TypeScript）：

```bash
npm install -g @modelcontextprotocol/sdk
```

最簡 server（TypeScript）：

```typescript
import { Server } from '@modelcontextprotocol/sdk/server';

const server = new Server({
  name: 'my-custom-mcp',
  version: '0.1.0',
});

server.setRequestHandler('tools/list', async () => ({
  tools: [{
    name: 'hello',
    description: 'Say hello',
    inputSchema: { type: 'object' }
  }]
}));

server.setRequestHandler('tools/call', async (req) => {
  if (req.params.name === 'hello') {
    return { content: [{ type: 'text', text: 'Hello from MCP!' }] };
  }
});

server.connect();
```

Build → 註冊 → Claude 可以叫起「hello」工具。

呢個框架等於「Claude Code 嘅外掛系統」—— 對自訂整合開放。

### 變化 3：團隊用 MCP —— 共用設定

每個團隊成員自己安裝 MCP，但設定可以共用：

`./.claude/settings.json` 入 git：

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

每個開發者喺自己 shell 設自己環境變數（`GITHUB_TOKEN`）。設定共用，憑證個別。

## 一個 mindset

MCP 嘅深層 insight：**AI 嘅未來唔係「更聰明嘅模型」，係「標準化嘅工具生態」**。

Claude 自己識推理但唔連工具。MCP = 嗰個連線層。每多安裝 1 個 MCP，Claude 嘅行動範圍擴闊。

5 個 MCP 設好 + Claude Code = 你嘅個人 AI 助手真實咁連到你每日工具鏈：
- GitHub for code
- Notion for 任務 / 文檔
- Slack for 通訊
- Postgres for 數據
- Filesystem for 檔案操作

Claude 變成「協調嘅嗰一層」vs「孤立嘅模型」。

最後提醒：

- ✅ **由 1-2 個 MCP 開始**，第一日唔好裝 10 個。逐步擴展。
- ⚠️ **MCP 工具通常 access 真實生產系統**。權限紀律好重要。Token / API key 永遠唔好 commit git。
- 🎯 **Build 自己嘅 MCP** 如果你公司內部工具冇公開版本。SDK 對中級開發者都易上手。

下個鐘安裝 GitHub MCP。1 個禮拜後你會本能咁「Claude，list 我哋啲 PR」。
