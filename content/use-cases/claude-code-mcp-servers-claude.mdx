## 情境

你 Claude Code 1 個月。Workflow:

- Day 1: 「Claude review my code」 ✓
- Day 7: 「Claude write commit message」 ✓
- Day 14: 「Claude refactor this function」 ✓
- Day 21: 「Claude check our Notion task board」 ❌

Claude alone 唔識 read Notion. 你 manual copy task → paste → ask Claude. Slow.

呢個 limitation 嘅 root cause：**Claude 嘅 default action 限於 read / write files + run shell + web search**. External SaaS service (Notion / Slack / GitHub API / database / etc) 需要 specific integration layer.

呢個 layer 就是 **MCP (Model Context Protocol)** — Anthropic 推 嘅 open standard，等 Claude (and other AI) 識 connect external tool 透 standardized way。

**Concept**:
- **MCP server** = 一個 process，expose specific tools to AI (e.g., "read Notion task", "create GitHub issue")
- **Claude Code** = consumer of MCP servers
- **You** = install MCP server + register with Claude

呢篇拆 5 個 most useful MCP server + 10-分鐘 install。

## 跟住做

### 1. MCP architecture quick visual

```
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Notion)] ←── API ──→ [Notion]
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Slack)]  ←── API ──→ [Slack]
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Postgres)] ←── SQL ──→ [Database]
```

Claude Code 1 個 user，可以 simultaneously connect 多 個 MCP server。Each server exposes specific tool set。

### 2. Top 5 MCP servers most useful

**1. GitHub MCP** — repo browsing, issue / PR management, code search

Use case:
- 「List open PR 喺 our repo」
- 「Read README from team's other project」
- 「Create issue 標題 X, body Y」

**2. Notion MCP** — task / doc / database read + write

Use case:
- 「Show our team's task board」
- 「Mark task X as done」
- 「Search docs for 'API key handling'」

**3. Slack MCP** — channel browsing, message send, search history

Use case:
- 「Search #engineering for discussion on Redis」
- 「Send daily standup to #team channel」
- 「Read DMs from last hour」

**4. Filesystem MCP** — multi-folder operations beyond Claude Code default

Use case:
- 「Compare structure of /project-a vs /project-b」
- 「Sync README between 5 sibling project」

**5. Postgres / SQLite MCP** — direct database query (read-only safe option exists)

Use case:
- 「Query our staging DB for users created last week」
- 「Show schema for `orders` table」
- 「Run 健康 check query」

(Other notable: Filesystem, Brave Search, Sentry, Linear, Stripe — community + official mix.)

### 3. Install your first MCP — GitHub MCP

GitHub MCP 通常 most universally useful。Install:

```bash
# Most MCP servers npm or pip
npm install -g @modelcontextprotocol/server-github
```

Configure GitHub token：去 [github.com/settings/tokens](https://github.com/settings/tokens), create personal access token with `repo` scope。Save token securely。

### 4. Register MCP server with Claude Code

Open Claude Code config:

```bash
claude --config
# 或 manual edit ~/.claude/settings.json
```

Add MCP server entry:

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "ghp_yourtokenhere"
      }
    }
  }
}
```

Save + 重 launch Claude Code:

```bash
claude
```

Claude Code load 時自動 detect MCP server。第一個 prompt 試：

```
List open PR 喺 hideyau28/my-good-friend-claude repo
```

如 setup OK，Claude 透 GitHub MCP query GitHub API → return PR list。

### 5. Verify MCP working

Inside Claude Code session, type:

```
/mcp
```

顯示 connected MCP servers 同 their tools。應 see「github」 列出 with tool 如 `list_pull_requests`, `create_issue`, etc.

如 唔 show: check error log
```bash
claude --verbose
```

### 6. Permission management

每 MCP tool 第一次 invoke，Claude Code ask permission:

- `Allow once`
- `Allow always for this MCP server`
- `Reject`

新手建議: read-only operations (list / search / read) → 「Allow always」 safe. 任何 mutate (create / update / delete) → manual approve each。

### 7. 試 second MCP — Notion

Install Notion MCP:

```bash
npm install -g @modelcontextprotocol/server-notion
```

Get Notion integration token: [notion.so/my-integrations](https://www.notion.so/my-integrations) → Create new integration → copy「Internal Integration Secret」

Add to Claude Code config:

```json
{
  "mcpServers": {
    "github": { ... },
    "notion": {
      "command": "mcp-server-notion",
      "env": {
        "NOTION_API_KEY": "secret_yourtokenhere"
      }
    }
  }
}
```

⚠️ 同樣，每個 Notion page / database 要 explicit「share with integration」 喺 Notion UI — 通常 用戶 forget 呢 step。No share = no access.

Restart Claude Code, test:
```
List task 喺 我 「Engineering 2026」 Notion database
```

## 變化

### 變化 1：Install community MCP from GitHub

唔 限 npm-published MCP。Community-developed MCPs 多 on GitHub:

```bash
# Clone community MCP repo
git clone https://github.com/user/their-mcp-server
cd their-mcp-server
npm install
```

Register with absolute path:

```json
{
  "mcpServers": {
    "custom": {
      "command": "/full/path/to/their-mcp-server/dist/index.js"
    }
  }
}
```

### 變化 2：Build your own MCP server

Anthropic 提供 MCP SDK (Python + TypeScript):

```bash
npm install -g @modelcontextprotocol/sdk
```

Minimal server (TypeScript):

```typescript
import { Server } from '@modelcontextprotocol/sdk/server';

const server = new Server({
  name: 'my-custom-mcp',
  version: '0.1.0',
});

server.setRequestHandler('tools/list', async () => ({
  tools: [{
    name: 'hello',
    description: 'Say hello',
    inputSchema: { type: 'object' }
  }]
}));

server.setRequestHandler('tools/call', async (req) => {
  if (req.params.name === 'hello') {
    return { content: [{ type: 'text', text: 'Hello from MCP!' }] };
  }
});

server.connect();
```

Build → register → Claude 可 invoke 「hello」 tool。

呢個 framework 等如「Claude Code 嘅 plugin system」 — open for custom integration。

### 變化 3：MCP for team — shared config

每 team member 自己 install MCP server，但 config 可以 shared:

`./.claude/settings.json` 入 git:

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

每 dev set own env var (`GITHUB_TOKEN`) 喺 own shell。Config file shared，credentials 個 別。

## 一個 mindset

MCP 嘅 deeper insight: **AI 嘅 future 唔係 「smarter model」, 係 「standardized tool ecosystem」**.

Claude alone 識 reason but 唔 connect tool 嘅 access. MCP = the connecting layer。每多 1 個 MCP server install, Claude 嘅 actionable scope 拓寬。

5 個 MCP setup + Claude Code = 你 personal AI assistant 真實 connected to 你 daily tool stack:
- GitHub for code
- Notion for task / docs
- Slack for communication
- Postgres for data
- Filesystem for file ops

Claude 變成 「the layer that orchestrates」 vs 「the model in isolation」.

最後 reminder：

- ✅ **Start with 1-2 MCP**, 唔 install 10 個 first day. Gradual expand stack.
- ⚠️ **MCP tools 通常 access real production system**. Permission discipline 真。Token / API key never commit git.
- 🎯 **Build own MCP** 如 你公司 internal tool 唔 publicly available. SDK approachable for medium-skilled dev.

下個 hour install GitHub MCP. 1 個禮拜 後你 daily 「Claude, list our PRs」 instinctively.
