## 情境

你 Claude Code 1 個月。Workflow:

- Day 1:「Claude review my code」✓
- Day 7:「Claude write commit message」✓
- Day 14:「Claude refactor this function」✓
- Day 21:「Claude check our Notion task board」❌

Claude alone 唔識 read Notion。你手動 copy task → paste → ask Claude。慢。

呢個 limitation 嘅根源：**Claude 嘅 default action 限於 read / write files + run shell + web search**。External SaaS service (Notion / Slack / GitHub API / database / etc) 需要特定嘅 integration layer。

呢個 layer 就係 **MCP (Model Context Protocol)** — Anthropic 推嘅 open standard，等 Claude (and other AI) 識用標準方式 connect external tool。

**概念**:
- **MCP server** = 一個 process，expose specific tools to AI (e.g., "read Notion task", "create GitHub issue")
- **Claude Code** = MCP server 嘅 consumer
- **你** = install MCP server + register with Claude

呢篇拆 5 個最有用嘅 MCP server + 10 分鐘 install。

## 跟住做

### 1. MCP architecture quick visual

```
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Notion)] ←── API ──→ [Notion]
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Slack)]  ←── API ──→ [Slack]
[Claude Code] ←── MCP protocol ──→ [MCP server (e.g., Postgres)] ←── SQL ──→ [Database]
```

Claude Code 1 個 user，可以同時 connect 多個 MCP server。每個 server expose 各自嘅 tool set。

### 2. Top 5 最有用嘅 MCP servers

**1. GitHub MCP** — repo browsing, issue / PR management, code search

Use case:
- 「List open PR 喺 our repo」
- 「Read README from team's other project」
- 「Create issue 標題 X, body Y」

**2. Notion MCP** — task / doc / database read + write

Use case:
- 「Show our team's task board」
- 「Mark task X as done」
- 「Search docs for 'API key handling'」

**3. Slack MCP** — channel browsing, message send, search history

Use case:
- 「Search #engineering for discussion on Redis」
- 「Send daily standup to #team channel」
- 「Read DMs from last hour」

**4. Filesystem MCP** — 跨 folder operation 超過 Claude Code default

Use case:
- 「Compare structure of /project-a vs /project-b」
- 「Sync README between 5 sibling project」

**5. Postgres / SQLite MCP** — 直接 query database (read-only safe option exists)

Use case:
- 「Query our staging DB for users created last week」
- 「Show schema for `orders` table」
- 「Run 健康 check query」

(其他 notable: Filesystem, Brave Search, Sentry, Linear, Stripe — community + 官方混合.)

### 3. Install 你第一個 MCP — GitHub MCP

GitHub MCP 通常最 universally useful。Install:

```bash
# 大部分 MCP servers npm 或 pip
npm install -g @modelcontextprotocol/server-github
```

Configure GitHub token：去 [github.com/settings/tokens](https://github.com/settings/tokens), create personal access token with `repo` scope。Save token 安全。

### 4. Register MCP server 落 Claude Code

打開 Claude Code config:

```bash
claude --config
# 或手動 edit ~/.claude/settings.json
```

Add MCP server entry:

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "ghp_yourtokenhere"
      }
    }
  }
}
```

Save + 重 launch Claude Code:

```bash
claude
```

Claude Code load 時自動 detect MCP server。第一個 prompt 試：

```
List open PR 喺 hideyau28/my-good-friend-claude repo
```

如 setup OK，Claude 透過 GitHub MCP query GitHub API → return PR list。

### 5. Verify MCP working

喺 Claude Code session 入面 type:

```
/mcp
```

顯示 connected MCP servers 同佢哋嘅 tools。應該見到「github」列出，附帶 `list_pull_requests`, `create_issue` 等 tool。

如唔 show: check error log
```bash
claude --verbose
```

### 6. 權限管理

每個 MCP tool 第一次 invoke，Claude Code ask permission:

- `Allow once`
- `Allow always for this MCP server`
- `Reject`

新手建議: read-only operations (list / search / read) → 「Allow always」安全。任何 mutate (create / update / delete) → 每次手動 approve。

### 7. 試 second MCP — Notion

Install Notion MCP:

```bash
npm install -g @modelcontextprotocol/server-notion
```

Get Notion integration token: [notion.so/my-integrations](https://www.notion.so/my-integrations) → Create new integration → copy「Internal Integration Secret」

Add 落 Claude Code config:

```json
{
  "mcpServers": {
    "github": { ... },
    "notion": {
      "command": "mcp-server-notion",
      "env": {
        "NOTION_API_KEY": "secret_yourtokenhere"
      }
    }
  }
}
```

⚠️ 同樣，每個 Notion page / database 要明確「share with integration」喺 Notion UI — 用戶通常會唔記得呢一步。冇 share = 冇 access。

Restart Claude Code, test:
```
List task 喺我「Engineering 2026」Notion database
```

## 變化

### 變化 1：Install community MCP from GitHub

唔限於 npm-published MCP。Community-developed MCPs 多數喺 GitHub:

```bash
# Clone community MCP repo
git clone https://github.com/user/their-mcp-server
cd their-mcp-server
npm install
```

Register with absolute path:

```json
{
  "mcpServers": {
    "custom": {
      "command": "/full/path/to/their-mcp-server/dist/index.js"
    }
  }
}
```

### 變化 2：Build 自己嘅 MCP server

Anthropic 提供 MCP SDK (Python + TypeScript):

```bash
npm install -g @modelcontextprotocol/sdk
```

Minimal server (TypeScript):

```typescript
import { Server } from '@modelcontextprotocol/sdk/server';

const server = new Server({
  name: 'my-custom-mcp',
  version: '0.1.0',
});

server.setRequestHandler('tools/list', async () => ({
  tools: [{
    name: 'hello',
    description: 'Say hello',
    inputSchema: { type: 'object' }
  }]
}));

server.setRequestHandler('tools/call', async (req) => {
  if (req.params.name === 'hello') {
    return { content: [{ type: 'text', text: 'Hello from MCP!' }] };
  }
});

server.connect();
```

Build → register → Claude 可 invoke「hello」tool。

呢個 framework 等如「Claude Code 嘅 plugin system」— open for custom integration。

### 變化 3：MCP for team — shared config

每個 team member 自己 install MCP server，但 config 可以 share:

`./.claude/settings.json` 入 git:

```json
{
  "mcpServers": {
    "github": {
      "command": "mcp-server-github",
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

每個 dev set 自己 env var (`GITHUB_TOKEN`) 喺自己 shell。Config file shared，credentials 個別。

## 一個 mindset

MCP 嘅深層 insight: **AI 嘅未來唔係「smarter model」，係「標準化 tool ecosystem」**。

Claude alone 識推理但唔 connect 到 tool。MCP = 個 connecting layer。每多 install 1 個 MCP server，Claude 嘅行動範圍拓寬。

5 個 MCP setup + Claude Code = 你嘅 personal AI assistant 真實 connected to 你 daily tool stack:
- GitHub for code
- Notion for task / docs
- Slack for communication
- Postgres for data
- Filesystem for file ops

Claude 變成「the layer that orchestrates」vs「the model in isolation」。

最後 reminder：

- ✅ **由 1-2 個 MCP 開始**，第一日唔好 install 10 個。Gradual 擴展 stack。
- ⚠️ **MCP tools 通常 access real production system**。Permission discipline 真重要。Token / API key 永遠唔好 commit git。
- 🎯 **Build 自己嘅 MCP** 如你公司 internal tool 冇 publicly available。SDK 對中等程度 dev 都易上手。

下個鐘 install GitHub MCP。1 個禮拜後你 daily「Claude, list our PRs」instinctively 咁講。
