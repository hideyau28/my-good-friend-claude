## 情境

你管緊一個 5 人 startup / 創意工作室。你訂閱緊：

- Notion ($10/月/人)
- Asana ($15/月/人)
- Linear ($10/月/人 — 工程師專用)
- Figma ($15/月/人)
- Slack ($8/月/人)
- 各人自己嘅訂閱（ChatGPT、Claude Pro、Canva⋯⋯）

5 人 × 6 個 SaaS = 每月 burn $300+。而且**功能重疊**（Notion 部分 cover 到 Asana、Asana 部分 cover 到 Linear）。

呢篇用 Claude 幫你做策略性對比 + 簡化工具組合。

## 跟住做

### 1. 列出你而家用緊 + 每個 tool 嘅實際用途

每個 tool：
- 每月開支（每人收費 × 人數）
- 主要用途（你哋實際用嚟做乜，唔係功能清單）
- 邊個 team 用、邊個 team 唔用
- 接受度（80% 每日用？30%？）

冇呢個老實評估，AI 出嘅結果會變行貨「呢個 tool 強 X、嗰個強 Y」嘅 marketing 分析。

### 2. 用呢個 prompt

<PromptBlock label="完整 prompt — SaaS 工具組合 audit">
我哋有 [N 人] team，訂閱緊以下 SaaS：

[List 每個 tool + cost + actual use case + adoption]

我哋嘅 priority（排優先次序）：

1. [例：team alignment + project visibility]
2. [例：個別 task 跟蹤]
3. [例：document / knowledge base]
4. [例：communication]

請整 4 個 section：

**Part 1：功能 overlap audit**
邊兩個 tool 我哋 use case 上 overlap？specific：
- 我哋而家 use Notion for X，Asana for Y——其實 [tool] 一個 cover 兩樣
- Migration cost: 由 [Tool A] 走去 [Tool B] 邊樣難（template / data export / team retrain）

**Part 2：Recommend simplification**
我哋 ideal stack（揀 3-4 個 tool max，cover 上面 priorities）+
- 邊個 tool keep
- 邊個 tool drop
- 邊個 tool migrate（A → B）

**Part 3：Cost impact**
我哋 simplification 之後嘅 monthly cost
Annualized savings
ROI period（含 migration time cost）

**Part 4：Risk + Mitigation**
- Adoption risk：team member resist switching
- Feature gap：drop tool 之後缺嘅 specific function 點 work around
- Migration timeline：建議 phased approach（唔好 big-bang 一日 change all）

語氣：opinionated（畀 recommendation，唔好「it depends」fence-sit）。
</PromptBlock>

### 3. 同 team 傾 + 小規模試用

唔好單方面決定。將 AI 嘅建議帶落 team weekly：

- 提出成本考量
- 討論簡化方案
- 試用：揀一個 sub-team 試 2 個禮拜新工具組合
- 評估接受度 + 阻力

呢類工具整合通常需要 6-8 個禮拜完成。唔好急。

## 變化例子

### 變化 1：你係 solo dev / freelancer

<PromptBlock label="變化 1 — Solo 工具組合 audit">
我係 solo（[freelance / dev / creator]）。
我嘅 SaaS 訂閱：[列出]

我嘅 needs：
1. [例：客戶 project management]
2. [例：personal task list]
3. [例：notes + reference]

Solo 情況唔同 team——個 friction 低 / decision speed 快 / adoption 100%。

請：
1. Recommend 一個 minimum viable stack（2-3 個 tool max）
2. 一個「all-in-one」嘅 single tool 選擇（Notion / Obsidian / 純 .md file in iCloud / Capacities）
3. 為 solo context 推薦邊種 approach（minimum viable vs all-in-one）
4. 月 cost 對比
</PromptBlock>

### 變化 2：用 Claude 取代某個 SaaS

<PromptBlock label="變化 2 — Claude 直接取代 SaaS">
我考慮用 Claude Chat / Project 取代 [某個 SaaS]。

例如：
- Notion AI → Claude Chat 直接寫 / brainstorm
- Grammarly → Claude polish
- 寫 social caption tools → Claude 一條 prompt

請：
1. 對比 Claude vs 該 SaaS 嘅 specific use case
2. Trade-off：Claude 嘅 leverage（深度）vs 該 SaaS 嘅 leverage（UI / workflow）
3. Decision framework：邊類 task Claude 真係 work，邊類仲係 keep 個 SaaS
4. Recommend：drop 邊個 SaaS 最 obvious win，邊個 marginal save 唔值得 disruption
</PromptBlock>

## 一個思維轉變

每年做一次 SaaS 工具組合審視。唔需要徹底大改造——目標係**搵出重疊 + 取消已經冇用嘅訂閱**。

平均情況：HK solo / 細 team 每年慳 $1,500-3,000 嘅 monthly burn。呢個係「隱形加薪」——同樣生產力，少咗固定開支。
