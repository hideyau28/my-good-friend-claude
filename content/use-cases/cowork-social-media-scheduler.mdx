## 情境

你係 solo creator / 小品牌老闆。每星期：

- 寫一篇 newsletter（300-800 字嗰種）
- 然後手動拆 5 條社交媒體貼文：
  - IG caption + hashtag
  - Threads
  - Facebook
  - LinkedIn
  - X thread

正常人要 1-1.5 個鐘先搞掂。跟住你又要排程落 Buffer / Later，遷就唔同時區再睇一次。

呢類分發工作根本係機械式。創作 newsletter 嘅獨特價值喺你度；分發 newsletter 嘅槓桿就喺自動化。

呢篇用 Cowork 設定：**每星期一 newsletter 發佈之後，自動拆 5 條社交貼文 + 排程落 Buffer 隊列**。

> ⚠️ Cowork 仲係研究預覽版。涉及社交帳號權限，劃最低權限嘅 OAuth。永遠係「排做草稿」唔係「即時發佈」——保留最後嗰下人手檢視。

## 跟住做

### 1. 建立 newsletter 來源

由 Beehiiv（或者你嘅 newsletter 平台）設定 webhook / RSS feed 落一個專用資料夾：

```
~/Documents/newsletter-published/
└── 2026-XX/
    └── YYYY-MM-DD-issue-N.md
```

每次發佈 newsletter，呢度會自動有 markdown 副本。

### 2. 用呢個 prompt 設定

<PromptBlock label="完整 prompt — 設定每週社交分發">
我想設定 Cowork 自動將我嘅 newsletter 拆做社交貼文。

**觸發條件：** 每星期 newsletter 發佈之後 4 個鐘（即星期日 1pm，避開正喺度發送嗰一刻）

**輸入來源：** ~/Documents/newsletter-published/ 嘅最新 .md 檔案

**每次觸發：**

**第一步：讀取 newsletter**
- 抽出 newsletter 嘅中心思想（1 句）
- 抽出 2-3 個支持觀點嘅引言 / 觀察
- 抽出 newsletter URL（最頂嘅中繼資料）

**第二步：生成 5 條平台原生貼文**

每個平台嘅具體規則：

A. **IG Caption**（300 字內）
- 吸引眼球嘅第一句
- 中段用 emoji 分段
- 結尾 CTA：「完整版睇 link in bio」
- 5-8 個 hashtag，混合大細 tag

B. **Threads post**（500 字內）
- 唔用 hashtag
- 分多段，對話語氣
- Link 放喺對話串回覆度（避免第一條貼文嘅 link 被限流）

C. **Facebook post**（400 字）
- 開頭有個人化嘅吸引點
- 敘事形式講故事
- 結尾 CTA + 問題

D. **LinkedIn post**（300 字）
- 有洞察力嘅開場白
- 3 個重點
- 結尾有深度嘅問題

E. **X thread**（5-7 條 tweet）
- 第一條 hook
- 中間講故事
- 最後 CTA

**第三步：草稿放入 Buffer 隊列**
- 將每條貼文推落對應平台嘅 Buffer 隊列
- 排程時間：
  - IG: 該週星期二 11am
  - Threads: 該週星期三 4pm
  - Facebook: 該週星期四 9am
  - LinkedIn: 該週星期二 8am
  - X thread: 該週星期三 10am
- 狀態：**草稿（等待檢視）**，唔係即時發佈

**第四步：電郵我檢視提醒**
- Subject：「Weekly Social Schedule Ready for Review — N posts queued」
- 內容：每個貼文嘅預覽 + Buffer link 直接去編輯
- 我有 24 小時檢視（星期日 5pm → 星期一 5pm）
- 星期一 5pm 之後貼文按排程自動發佈

**限制：**
- 唔好自動發佈——只可以排隊做草稿
- 一旦我編輯過，就唔好再改個 hashtag / 內容
- 私隱：唔好喺社交貼文提及任何內部團隊 / 客戶名（除非 newsletter 已經公開提過）

設定好之後確認，再排程。
</PromptBlock>

### 3. 第一個禮拜仔細人手檢視

第一次輸出通常 80% OK，20% 要微調：

- Hashtag 相關度（Claude 嘅預設 hashtag 可能過時）
- LinkedIn 語氣（容易太「Threads 味」）
- IG hook（第一句嘅攔截力）

第二個禮拜開始，檢視時間由 30 分鐘減到 10 分鐘。

### 4. 月尾評估互動率

每個月尾檢視 Buffer 數據分析：
- 邊個平台嘅互動率強過基準
- 邊種貼文格式 AI 持續出得差（要人手重寫）

改良 prompt → 改善輸出 → 減少人手編輯。通常 3-4 個月，就會去到一個合理、近乎自動運作嘅狀態。

## 變化例子

### 變化 1：Podcast 而唔係 newsletter

<PromptBlock label="變化 1 — Podcast → social">
我嘅源頭係 podcast，唔係 newsletter。
每集發佈之後我想要類似嘅分發：

- 用 podcast 逐字稿做來源
- 拆 3-4 個引言圖（IG carousel slide）
- 拆 1 個 60 秒嘅短片講稿（reel / shorts）
- LinkedIn 公佈貼文

請修改流程去配合 podcast 工作流程。
</PromptBlock>

### 變化 2：品牌老闆嘅產品發佈公佈

<PromptBlock label="變化 2 — 產品發佈公佈">
我有產品發佈公佈（[描述]）。同 newsletter 嘅每週排程唔同，呢個係一次性嘅。

請：
1. 拆份公佈做發佈日錯開時間嘅貼文（唔同平台唔同時間）
2. 加預熱貼文（發佈前 3 日）
3. 加發佈後跟進（發佈後 1 個禮拜收集反應）
4. 日曆上同步落 Buffer 做全部草稿（等我可以檢視時間同訊息）
</PromptBlock>

### 變化 3：你負責企業品牌 / 公司通訊

<PromptBlock label="變化 3 — 公司通訊">
我做緊 [公司] 嘅 marketing / comms。我嘅來源係內部公司 blog（[平台]）。

請：
1. 跨平台發佈，但保持品牌語氣一致
2. 自動 tag 公司官方帳號（IG @companyname、LinkedIn company page）
3. 合規檢查：發佈之前標示任何分享機密 / 非公開資訊嘅內容
4. 必須人手批准：每個平台第一條貼文我要人手批准先發佈

呢個用途風險大（聲譽風險），預設要保守。
</PromptBlock>

## 拆解：點解會 work，同點解會爆

跟到上面就已經用得。下面呢段係畀**想由「跑一次 demo OK」做到「跑足一年都信得過」**嘅人——初學者可以跳過，唔影響你跟住做。

自動化最唔老實嘅地方係：**第一次跑靚仔，唔代表第十次冇事**。呢個排程流程，實際會喺呢幾個位爆，你要預咗：

**1. 觸發器抓錯文件（最常爆）**
prompt 叫 Cowork 讀「最新 .md 檔案」，但你個資料夾可能有草稿、有更正版、有上週嗰份。「最新」係以修改時間計定發佈時間計？如果你 publish 之後修正過一次 newsletter，Cowork 可能會重新處理舊文件。
- 會爆成點：同一篇 newsletter 出兩套貼文，或者拆咗一份未完成嘅草稿落 Buffer。
- 點救：命名規則加埋發佈狀態（例如 `YYYY-MM-DD-issue-N-published.md`）；prompt 指定「只處理檔名含 published 嘅最新一份」。每次觸發之後 Cowork 將該檔案移去 `processed/` 資料夾，避免重複處理。

**2. Buffer API 靜默失敗（最難察覺）**
Buffer 嘅 API 有時會回傳 200 OK 但其實貼文冇入到隊列——常見原因係 OAuth token 過期、帳號連線斷咗、或者你個 Buffer plan 已達排程上限。Cowork 唔一定識分辨「成功排程」同「server 話收到但其實唔會發佈」。
- 會爆成點：你收到「5 條貼文已排程」嘅電郵提醒，但 Buffer 入面係空的，星期二冇嘢出。
- 點救：prompt 加一步「排程之後即刻讀返 Buffer scheduled queue 確認條數」；同埋喺 Buffer 本身設定「排程成功」通知 email，獨立於 Cowork 嘅提醒。

**3. 24 小時審視窗靜默過去**
流程設定係「提醒電郵 → 24 小時內人手批准 → 自動發佈」。但如果你嗰週出差、電郵冇收到、或者提醒落咗垃圾桶，貼文會靜默地按排程出去——你睇都未睇過。
- 會爆成點：一條語氣唔啱、或者攞咗敏感客戶名嘅 LinkedIn 貼文靜靜地發咗出去。
- 點救：Buffer 嘅貼文一律設成「需要人手批准」（Approval Required）而唔係「草稿」，係兩個唔同嘅狀態——草稿到時間會自動發，Approval Required 唔批就唔會出。唔好依賴 Cowork 呢邊嘅「等我檢視」邏輯，喺 Buffer 平台本身做最後一道鎖。

**4. 平台語氣跨污染**
同一個 prompt run 生成 5 條貼文，Claude 有時會喺同一份「創意情緒」入面完成晒全部——結果 LinkedIn post 帶咗 Threads 嗰種碎片對話語氣，或者 IG caption 冇咗 emoji 嘅節奏感。
- 會爆成點：LinkedIn 聽落係「今日有個好有趣嘅發現……」，唔夠專業；IG 係一大段文冇分段、冇 hook。
- 點救：prompt 入面每個平台之後加一行「語氣 benchmark」——例如「LinkedIn：《Harvard Business Review》嘅洞見感，唔係 Twitter 線程」；同埋要求 Claude 每個平台獨立起草，唔好喺心裏面「改寫上一條」。月尾評估嗰步，如果某個平台輸出持續質素差，單獨抽出嚟用另一個更具體嘅 prompt 處理。

**5. Hashtag 過時或者相關度低**
Hashtag 係社交媒體入面最快過期嘅嘢。Claude 嘅訓練數據截止某個時間點，唔知道今個月哪條 tag 真係有人睇、哪條已經被 shadowban 淹沒。
- 會爆成點：IG 出咗一堆 2022 年嘅通用大 tag（`#contentcreator`、`#growthmindset`），觸及率極低；或者用咗個平台已經限流嘅 tag。
- 點救：每季人手更新一次 prompt 入面嘅 hashtag seed list，用你個帳號真實表現好嘅 tag 做參考；prompt 寫明「hashtag 限 8 個，混合 1-2 個細眾 niche tag + 2-3 個貼題中眾 tag，唔好出純粹大眾 tag」。

呢幾個位，就係「跑一次 demo OK」同「跑足一年都信得過」之間嘅距離。

## 一個思維框架

社交分發嘅腦力消耗比想像中高：

- 「我要切換思維模式由寫長文 → 寫 IG caption」
- 「我要諗 hashtag」
- 「我要排程時間」
- 「我要登入 Buffer」

每樣獨立睇好小事，但累積起上嚟會冇咗兩個鐘 + 一個清爽嘅週末朝早。

Cowork 嘅槓桿唔止係慳時間，係 **「我嘅創作精力留返落 newsletter 本身」**——分發變成後台程序。

最重要嘅提醒：**永遠唔好畀 Cowork 自動發佈**。自動排隊 + 人手檢視 + 人手批准 — 呢 3 步永遠要保持。
