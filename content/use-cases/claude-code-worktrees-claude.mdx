## 情境

你 mid-day 做緊 feature A:「Add user search to dashboard」。編寫 4 個 file, 30% done, 個腦個 model 已 loaded 晒。

老闆 ping:「Need 急 fix feature B production bug — users can't login since 2pm」。

Pre-worktree workflow:

1. Stash uncommitted A work: `git stash` (但 lose Claude Code session context)
2. Switch branch: `git checkout main && git pull && git checkout -b fix/login-bug`
3. Open Claude Code re-onboard:「我而家 fix login bug, 你 context 唔知 feature A。Forget that.」
4. Fix bug, commit, push, PR
5. Switch back: `git checkout feature-a`
6. Stash pop: `git stash pop`
7. Re-onboard Claude Code:「Resume feature A。我之前 stash, 而家 restored。」

Context switch overhead: 20-30 分鐘每個方向。Daily 你做 2-3 個 switch = 1-1.5 hr 純 switching 都蝕咗。

**Git worktree** 解決:

- 每個 branch 住喺**自己 folder**
- 你可以同時有 main、feature-a、fix-login 3 個 folder
- 每個 folder 自己 Claude Code session, 自己 context
- 換「project」= `cd ~/projects/feature-a` (零 overhead)

呢篇拆 worktree setup + Claude Code 整合 pattern。

## 跟住做

### 1. Worktree 概念

Standard git: 1 repo = 1 folder, branches share folder (只可以 1 個 active 一次)。

Worktree git: 1 repo = main folder + N worktree folders, each worktree = 1 個 active branch。

```
~/projects/my-app/                 ← main repo, branch=main
~/projects/my-app-feature-a/       ← worktree, branch=feature/user-search
~/projects/my-app-fix-login/       ← worktree, branch=fix/login-bug
```

3 個 folder，3 個 branch 並存。

### 2. Setup 第一個 worktree

去你 existing repo:

```bash
cd ~/projects/my-app
git status
# 確認 clean (no uncommitted change)
```

Create worktree:

```bash
# Format: git worktree add <path> <branch-name>
git worktree add ../my-app-feature-search feature/user-search
```

呢個 command:
- 創建 folder `~/projects/my-app-feature-search/`
- 喺該 folder check out branch `feature/user-search` (如唔 exist, create)
- Sub-folder share same `.git` 目錄 (via reference)

`ls ~/projects/`:
```
my-app/
my-app-feature-search/
```

Cd 入新 worktree:
```bash
cd ../my-app-feature-search
git status
# On branch feature/user-search
```

獨立 working directory，own state。

### 3. 喺 worktree 開 Claude Code

```bash
cd ~/projects/my-app-feature-search
claude
```

Claude Code 視呢個 folder 為自己嘅 project。Independent conversation history。No bleed from main repo。

開始 work feature:
```
我 implementing user search. 需要 add API endpoint /search?q= + frontend search bar.
```

Claude 跟你 feature 嘅 design + implement，independent context。

### 4. 突發 production bug — 開第二個 worktree

老闆 ping production bug。你唔 stash feature work, 唔 switch branch:

```bash
cd ~/projects/my-app
git worktree add ../my-app-fix-login fix/login-bug

cd ../my-app-fix-login
claude
```

新 Claude Code session, fresh context for fix:
```
急 fix: users can't login since 2pm. Help me investigate.
```

Claude 1 session: feature work, 冇 context 污染。
Claude 2 session: bug fix, fresh focus。

兩個同時 active。Tab 1 IDE + Claude Code A。Tab 2 IDE + Claude Code B。

### 5. Fix + ship + return

```bash
# In fix worktree
git status  # See your fix
git add .
git commit -m "fix: login bug, regenerated session tokens"
git push -u origin fix/login-bug
gh pr create
```

PR open, fix shipped。

回 feature worktree:
```bash
cd ~/projects/my-app-feature-search
# Feature work 仍然 intact, Claude Code session 仲 running
```

你 1 秒內 resume。冇 stash pop, 冇 re-onboard。

### 6. Merge 完 cleanup worktree

PR merged。Cleanup:

```bash
cd ~/projects/my-app
git worktree remove ../my-app-fix-login
# Removes folder + cleans worktree registry
git branch -d fix/login-bug
```

Folder 冇咗, Git history 完整 (commit merged into main)。

### 7. 常用 worktree commands

```bash
# 列嗮 worktrees
git worktree list

# Add new worktree
git worktree add <path> <branch>

# Remove worktree (clean exit)
git worktree remove <path>

# Force remove (uncommitted changes)
git worktree remove --force <path>

# Prune (cleanup stale registry entries)
git worktree prune
```

## 變化

### 變化 1：Claude Code parallel agent + worktree 組合

Claude Code 嘅 `Agent` tool 配 `isolation: "worktree"` parameter — spawn subagent 喺 separate worktree:

```
Agent({
  description: "Refactor auth module",
  isolation: "worktree",
  prompt: "Refactor /src/auth/* for clearer separation of concerns. Run all tests after."
})
```

Effect: Claude 自動 spawn subagent → create temp worktree → subagent work in isolation → 如 changes made, commit + 返 path。你 review → merge or discard。

呢個 pattern：「Claude experiments in isolated branch」+「if good, merge to main」。Risk-free exploration。

### 變化 2：5+ parallel worktrees pattern

進階用法: 5+ worktrees 並存。

```
~/projects/my-app/                  ← main
~/projects/my-app-feature-a/        ← feature A
~/projects/my-app-feature-b/        ← feature B
~/projects/my-app-experiment/       ← experimental
~/projects/my-app-hotfix/           ← hotfixes
~/projects/my-app-docs/             ← documentation rewrite
```

每個 worktree 自己 Claude Code session, 自己 task。你 daily:

- Morning: feature A worktree 1 hr
- Mid-morning: experiment worktree explore approach
- Lunch
- After lunch: feature B
- 3pm: docs rewrite
- 4pm: review hotfix

Day end: 5 個 PR 可能。Comparator: 傳統 1-branch workflow, 你最多 ship 1-2。

⚠️ Cognitive overhead: 全日記住 5 個 features。唔啱 less experienced dev。Sweet spot: 大部分人 2-3 parallel。

### 變化 3：Worktree + IDE tab 配對

Set IDE (VS Code) 多 window pattern:

- Window 1: main repo (overview / config)
- Window 2: feature-a worktree (focused work)
- Window 3: bug-fix worktree (急 task)

VS Code「Open Recent」/「Workspace」feature 管理。每個 window 嘅 file tree 只 show 該 worktree 嘅 files — focus 清晰。

每個 window 一個 Claude Code session。總共：3 個 IDE + 3 個 Claude = 1 個 dev orchestrate level beyond traditional。

## 一個 mindset

Worktree 嘅深層 insight: **Context switch overhead 唔係「inevitable」— 係「default workflow 嘅 friction」**。

傳統 git: 1 個 working dir, branches 係 view。Switch view = mental flush + reload。你個腦冇 capability of perfect mental flush — 你 spend 5-10 分鐘 re-load context。

Worktree: branches 係 separate 物理 workspaces。Switch = `cd`。個腦唔需要 flush — feature A space 仍然完整 ready when 你 return。Switch overhead → 近乎零。

呢個 paradigm 對 senior dev / consultant / multi-project freelancer 顯著改變 daily output。

最後 reminder：

- ✅ **先 try 2 個 worktree**。Comfort 多咗再加。
- ⚠️ **Folder 數量紀律**。>5 worktrees 個腦就會 track 唔到邊個係邊個。用 descriptive folder name。
- 🎯 **Worktree + Claude Code subagent + slash command = ultimate parallelism**。Senior 級 workflow。

下個大 feature 試 dedicate 一個 worktree。1 個禮拜後你 context switch friction 顯著降低。1 個月後你會 wonder 點解傳統 branch switch 一直係 default。
