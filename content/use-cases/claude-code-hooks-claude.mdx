## 情境

你 Claude Code 用咗 6 個禮拜：

- 尋日突發：Claude 改咗大 refactor，你手動 `git commit` —— 但唔記得跑 `npm run format`。CI lint 肥咗，要 amend commit revert + 排版 + 重新 push。額外 15 分鐘麻煩。

- 上禮拜：你測試一個破壞性 script，唔小心同意咗 Claude「跑 rm -rf node_modules」（本意係清理）—— 但 Claude 跑去 `rm -rf /tmp/某個重要 folder`。Recovery 1 個鐘。

- 上個月：你 push 咗一個 commit，後來先發覺你 import 咗 production API key 入 .env.example。出醜大鑊洩漏。

呢類錯誤共通 pattern：**靠手動紀律**。你個腦應該記得每次做 X（排版 / 驗證 / 掃描），但個腦偶爾會唔記得 = 錯誤漏出去。

**Hook** 解決：Claude Code 執行週期入面特定事件你可以 hook 入去 —— 自動跑你要嘅檢查 / 動作。

**3 個關鍵事件**：

- **PreToolUse**：Claude 用工具前（read / write / bash / 等）—— 可以檢視 + 阻擋
- **PostToolUse**：工具跑完後 —— 可以反應 / 串連
- **Stop**：session 結束 —— 最後驗證 / cleanup

呢篇拆 5 個必備 hook + setup 步驟。

## 跟住做

### 1. Hook 架構

Hook = 一個 shell script 你喺 Claude Code settings.json 註冊。每次特定事件觸發，Claude Code 自動跑你 script + 按 script 嘅 exit code 決定動作。

```
[Claude 動作: write file] 
   ↓
[PreToolUse hook fires] 
   ↓ (exit 0: 准 / exit non-0: 阻)
[Tool 跑] 
   ↓
[PostToolUse hook fires] 
   ↓
[下一個動作]
```

### 2. 喺 settings.json 設 hook

打開設定：

```bash
claude --config
# 或手動 edit ~/.claude/settings.json
```

Hook 結構：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/pre-bash-safety.sh"
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/hooks/post-write-format.sh"
          }
        ]
      }
    ]
  }
}
```

- `matcher`：工具名（Bash / Write / Read / Edit / 等）—— hook 只係 matching 工具嗰陣觸發
- `command`：shell script 路徑

### 3. Hook 1 —— 阻破壞性指令（安全）

建立 `~/.claude/hooks/pre-bash-safety.sh`：

```bash
#!/bin/bash
# Read Claude's intended command from stdin
input=$(cat)
command=$(echo "$input" | jq -r '.command')

# 列 danger pattern
declare -a danger_patterns=(
  "rm -rf /"
  "rm -rf ~"
  "rm -rf \$HOME"
  "dd if=/dev/"
  "mkfs"
  "shutdown"
  ":(){:|:&};:"  # fork bomb
  "chmod -R 777 /"
  "git push --force.*main"
  "git push --force.*master"
)

# Check pattern
for pattern in "${danger_patterns[@]}"; do
  if echo "$command" | grep -qE "$pattern"; then
    echo "🚫 BLOCKED: command matches danger pattern '$pattern'" >&2
    exit 1
  fi
done

exit 0
```

`chmod +x ~/.claude/hooks/pre-bash-safety.sh`

註冊 hook（按 Step 2 例子）。

效果：如 Claude 試 `rm -rf /`（真嘅或者唔小心），hook 阻擋 + Claude 見到錯誤 → 試其他做法。

### 4. Hook 2 —— Write 完自動排版

建立 `~/.claude/hooks/post-write-format.sh`：

```bash
#!/bin/bash
input=$(cat)
file_path=$(echo "$input" | jq -r '.tool_input.file_path')

# 確認係 code file
if [[ "$file_path" =~ \.(ts|tsx|js|jsx|json|md)$ ]]; then
  cd "$(dirname "$file_path")"
  # Use prettier if installed
  if command -v prettier >/dev/null; then
    prettier --write "$file_path" 2>/dev/null
  fi
fi

exit 0
```

`chmod +x` + 註冊 PostToolUse for Write matcher。

效果：Claude 寫新 file / 改現有 file → prettier 自動跑 → 你永遠唔會見到排版 lint error。

### 5. Hook 3 —— Stop session 時 cleanup

建立 `~/.claude/hooks/stop-cleanup.sh`：

```bash
#!/bin/bash
# Auto-cleanup .DS_Store + temp file at session end
find . -name ".DS_Store" -delete 2>/dev/null
find . -name "*.tmp" -delete 2>/dev/null

# Check for any uncommitted changes
if [ -n "$(git status --porcelain 2>/dev/null)" ]; then
  echo "⚠️  Uncommitted changes detected" >&2
fi

exit 0
```

註冊做 Stop hook：

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [{
          "type": "command",
          "command": "~/.claude/hooks/stop-cleanup.sh"
        }]
      }
    ]
  }
}
```

效果：每次你退出 Claude Code session → cleanup 跑 → git 工作目錄永遠唔會留低 cruft。

### 6. Hook 4 —— Commit 前安全檢查

建立 `~/.claude/hooks/pre-bash-precommit.sh`：

```bash
#!/bin/bash
input=$(cat)
command=$(echo "$input" | jq -r '.command')

# Detect git commit attempt
if [[ "$command" == "git commit"* ]]; then
  # Check for hardcoded secrets in staged files
  if git diff --staged | grep -qE '(api[_-]?key|secret|password)\s*=\s*["'"'"'][^"'"'"']{8,}'; then
    echo "⚠️  Potential secret detected in staged file" >&2
    echo "Review with: git diff --staged | grep -iE 'key|secret|password'" >&2
    exit 1
  fi
  
  # Check for debug code
  if git diff --staged | grep -qE 'console\.log\(|print\(.*DEBUG'; then
    echo "⚠️  Debug console.log/print left in staged files" >&2
  fi
fi

exit 0
```

註冊 PreToolUse for Bash matcher（同 safety 一齊）。

效果：Claude 試 `git commit` → hook 檢查 staged 內容 → 若 detect 到 secret → 阻 + 建議 review。

### 7. Hook 5 —— Code 改完自動 test

建立 `~/.claude/hooks/post-write-test.sh`：

```bash
#!/bin/bash
input=$(cat)
file_path=$(echo "$input" | jq -r '.tool_input.file_path')

# Run tests if source code change
if [[ "$file_path" =~ /src/.+\.(ts|tsx|js|jsx)$ ]]; then
  if [ -f "package.json" ]; then
    npm test --silent 2>&1 | head -20
  fi
fi

exit 0
```

效果：Claude 改 src file → test 自動跑 → 結果顯示畀 Claude → Claude 若 fail 即時反應。

⚠️ 唔適合大 test suite（慢）。對大 project，限制 specific test pattern 或者跳過。

## 變化

### 變化 1：Project-scope hook

User-level hook 喺 `~/.claude/settings.json` 適用全部 project。Project-scope hook 喺 `./.claude/settings.json` 只適用呢個 project：

```json
// .claude/settings.json in project root
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{
          "type": "command",
          "command": "./.claude/hooks/project-specific.sh"
        }]
      }
    ]
  }
}
```

用途：
- 某 project 嚴格 lint 規則
- 某 project 唔想自動排版（legacy code）
- 某 project 具體 deploy 指令需要額外守門

### 變化 2：條件式 hook（事件過濾）

進階 hook 可以喺動作前檢視事件細節：

```bash
#!/bin/bash
input=$(cat)
file_path=$(echo "$input" | jq -r '.tool_input.file_path')

# Only fire for production code (skip test files)
if [[ "$file_path" =~ /test/ ]] || [[ "$file_path" =~ \.test\. ]]; then
  exit 0
fi

# Specific check for prod code
# ... rest of script
```

呢個粒度等你 hook 只觸發真正相關嘅情況。

### 變化 3：Hook + Skills + Commands 三者組合

3 層自訂 stack：hooks + skills + slash commands。

- **Hooks**：被動自動（事件觸發）
- **Skills**：workflow 邏輯（自動偵測意圖）
- **Commands**：用戶明確觸發

例：
- `/deploy` command 觸發 → spawn 4 個 subagent（並行檢查）→ 每個 subagent 寫 file → PostWrite hook 自動排版 → 所有 deploy gate 過 → 你 1 個字搞掂。

呢個架構複雜度高但每日 workflow 槓桿大。

## 一個 mindset

Hooks 嘅深層 insight：**AI agent + 人類協作嘅安全 + 質素唔應該全靠手動紀律。將煩悶嘅 guardrail 自動化**。

你個腦容量應該用喺判斷（邊個 design 較好、邊個 trade-off 該揀），唔好用喺記憶（跑排版 / 跑 test / 檢查 secret）。Hook 將後者交畀系統。

呢個 reframe：hook 唔係「微管理 Claude」，hook 係「自動化我自己嘅紀律」。Claude 動作 OK → hook 靜靜地。Claude 動作有問題 → hook 攔截 + 防止我會做嘅錯誤。

最後提醒：

- ✅ **由 1-2 個 hook 開始**（安全 + 排版）。每月加 1 個，當你發現重複嘅手動動作。
- ⚠️ **Hook script 嘅 bug 可以 block 全部 Claude 動作**。隔離測試 hook 先依賴佢。
- 🎯 **將團隊層嘅 hook commit 入 git repo**。Commit 前 secret scan 應該係全 team。
