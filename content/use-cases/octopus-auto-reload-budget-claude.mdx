## 情境

5 月 26 日夜晚，你 review 信用卡 statement。

過去 3 個月八達通自動增值（AAVS）：
- 4 月：HK$1,500（3 次 × HK$500）
- 3 月：HK$2,000（4 次 × HK$500）
- 5 月：HK$1,250（已增 + 預測）
- 平均：HK$1,580 / 月

問題：你張 AAVS 連住 HSBC Visa Signature。Cashback 0.4%。3 個月返 HK$19。

同事 Karen 同你講：「我用 Citi Cash Back，八達通 reload 1.2%，仲有迎新 HK$500。」

你心諗：「我用咗呢張卡 6 年，從來冇諗過 reload 邊張啱啲。」

仲有第二個問題：你部分 reload 係公司報得（出差搭港鐵），部分係私人（返工 + 出街食飯）。但 statement 全部撈埋一齊，每月 reconcile 要半個鐘 manually 翻 receipt。

第三個問題：八達通 app 嘅 transaction history 你由 2024 年都未睇過。可能有：
- 重複扣費（閘機 double-tap）
- 7-11 / OK 便利店嘅「細」衝動
- 出咗街但唔記得邊度洗咗 HK$80

呢個 admin 漏洞嘅悖論：
- **「八達通好細數」** —— 但年計 HK$18-30k 流走
- **「Cashback 1% 差別好細」** —— 但 HK$20k × 1% = HK$200 / 年免費
- **「Auto reload 方便」** —— 但你冇諗過邊張卡 reward 高
- **「公私數一齊撈」** —— 但月尾 reconcile 你 30 分鐘×12 = 6 個鐘 / 年

你真正需要：**1 次 setup（20 分鐘）→ 揀啱卡 + 自動 split + 月尾 audit prompt template**。

呢篇用 Claude **比較 5 張 popular HK 卡 reward + 設 reload threshold + 公私數 categorization + 月底 transaction audit**。

### 3. Auto reload + categorization rule setup

揀好卡之後，setup 自動規則：

<PromptBlock label="完整 prompt — AAVS threshold + 公私數 split 規則">
我已揀好[卡名]做 AAVS。

📐 **Threshold 設置問題**

八達通自動增值有 3 個常見 threshold：
- 餘額 ≤ HK$0 → reload HK$250
- 餘額 ≤ HK$0 → reload HK$500
- 餘額 ≤ HK$0 → reload HK$1,000

請幫我諗：
- 我每月 reload [金額]，邊個 threshold 最 optimal？
- Trade-off：細 reload 次數多（卡 reward 上限風險）vs 大 reload 次數少（八達通蝕咗實 cash）
- 出 trip / 大件事如何 handle（過 HK$3,000 上限）

🏷 **公私數 split 規則**

我想月尾分得清邊啲係：
- ✅ 公司可報（出差 / 客戶 meeting / 公幹）
- 🏠 私人開支（返工日常 / 出街）

但八達通 transaction history 唔會自動分。請建議：

1. **物理 split 法**
   - 用 2 張八達通？（1 張公幹 + 1 張私人）
   - Trade-off：易分 vs 銀包重 / 增值麻煩

2. **時間 split 法**
   - 公幹日子標記（calendar）
   - 月尾 cross-reference

3. **金額 pattern split 法**
   - 大額（多過 HK$30 一筆）通常 = 出 trip 港鐵 / 高鐵 / 機場
   - 細額（少過 HK$20）通常 = 7-11 / OK / 便利店 / 茶餐廳
   - 中額（HK$20-30）= 港鐵日常返工

4. **app rule 法**
   - 八達通 app 可標記交易（手動）
   - MTR Mobile app 紀錄獨立可見
   - 邊個 efficient？

🎯 **建議混合方案**

畀我 1 個適合「每月 80% 私人 + 20% 公幹」用戶嘅 hybrid system：
- 5 分鐘 / 月 admin 上限
- 數字準確度足夠報公司
- 唔需要每筆手動分類

📋 **Setup checklist**

- AAVS 申請 step
- 八達通 app 設置
- Calendar 提醒（公幹日 + 月尾 review）
- 銀行 alert（reload 異常觸發，例如 1 日 reload 3 次 = 唔正常）

語氣：實際操作 step，唔好理論。
</PromptBlock>

### 4. 月底 transaction audit prompt

每月最後一日，跑呢個 audit：

<PromptBlock label="完整 prompt — 月尾八達通 transaction audit">
我貼上個月八達通 transaction history（由 app export 或截圖貼字）：

[貼你嘅 transaction list — 通常 30-80 行]

格式：日期、時間、地點、金額、餘額

請分析：

📊 **Section 1：總支出分解**

- 總 reload 次數 + 金額
- 總消費次數 + 金額（reload 唔計）
- 平均每日 spend
- 同上 2 個月對比（若我貼埋）

📊 **Section 2：類別自動分類**

根據商戶名 / 地點推斷：
- 🚇 港鐵（搭車）
- 🚌 巴士 / 小巴
- 🏪 便利店（7-11 / OK / 759 / Daily Mart）
- 🍱 茶餐廳 / 食肆
- 🛒 超市（百佳 / 惠康）
- ☕ 連鎖 cafe（Starbucks / Pacific Coffee）
- 🅿️ 停車場
- 🎫 其他

每類：金額 + 次數 + 佔總 %

📊 **Section 3：公私數預估 split**

根據 pattern（時間 + 地點 + 我貼嘅公幹日 list）：
- 公幹估算：HK[金額]（[次數] 筆）
- 私人估算：HK[金額]（[次數] 筆）
- 唔肯定（需手動 review）：[筆數]

📊 **Section 4：異常 flag**

🚨 識別：
- 同地點 1 分鐘內重複扣費（閘機 double-tap）
- 異常大額（>HK$50 一筆）
- 凌晨時段消費（係咪你？）
- 7-11 / OK 高頻（細額衝動 pattern）

📊 **Section 5：spending insight**

- 「我嘅天我喺 7-11 洗咗 HK[金額]」嘅時刻
- 邊日 spending 最高（pattern：星期五晚？）
- 邊個地點重複出現（潛在自動消費）

📋 **Section 6：下個月行動**

- 1 個 cut 建議（最低 hanging fruit）
- 1 個 categorization 改善（pattern 唔清嘅地方）
- 1 個 admin 簡化（如果你發現我做緊重複嘢）

語氣：偵探式、數據為本、唔批判。
</PromptBlock>

15 分鐘搞掂，比手動翻 receipt 快 3 倍。

### 變化 2：學生家長 — student card + 學校貨幣

<PromptBlock label="變化 2 — 學生 / 家長 八達通 setup">
我有[細路年齡]，佢用學生八達通。學校仲有獨立貨幣（例如 e-card 食飯 / 圖書館）。

📐 **學生八達通專項**

- ✅ 港鐵半價（要 register）
- ✅ 部分巴士優惠
- ❌ AAVS 連家長信用卡（要家長帳戶授權）

🎯 **問題**

1. 應唔應該畀細路 AAVS？
   - Pro：唔會冇錢搭車
   - Con：失控 spending（7-11 零食）

2. 細路 spending 監控
   - 八達通 app 可分享 view
   - 每月 budget 例如 HK$300 零用？
   - Threshold 設低（reload HK$100 一次 vs HK$500）

3. 學校 e-card / 食飯 card
   - 同八達通分開
   - 月尾 reconcile（學校 portal vs 八達通）
   - 細路 spending literacy 教學機會

📊 **建議 setup**

- Reload threshold：[建議金額]
- 每月家長 review：5 分鐘 + 細路 5 分鐘一齊
- 異常 alert（1 日 >HK$80 觸發 notification）
- 「零食 budget」獨立 → 細路自己管

🚨 **唔好**
- AAVS 上限調高（細路無上限 = 災難）
- 唔同細路傾就斬 reload（失去學習機會）
- 用 monitoring 做罰（變鬥智鬥力）

語氣：細路 financial literacy 教學 +家長控制平衡。
</PromptBlock>

## 一個 mindset

八達通呢類「細數 admin」嘅悖論：**單筆 HK$50 唔覺，但年計 HK$20k 唔睇。Cashback 1% 差別好細，但年計 HK$200 免費錢。**

呢個悖論嘅原因：HK 人對「日常細額」有 cognitive blind spot。你會比較 HK$50 vs HK$80 嘅一頓飯，但唔會比較 0.4% vs 1.2% AAVS reward。

呢個 mindset 嘅 reframe：八達通唔係「方便工具」—— 係**「每月 HK$1-2k 嘅 mini budget category」**。佢值得 20 分鐘 / 年 setup + 15 分鐘 / 月 audit。Total 3 個鐘 / 年 → 慳 HK$300-500 cashback + 6 個鐘公私數 reconcile = 真係抵。

Claude 喺呢度嘅角色：你一個人 google 5 張卡 reward 要 1 個鐘 + 計上限 + 對自己 spend 估算 = 太煩。Claude 一次過比較 + 計 + 出 recommendation + 出 audit template。

最後提醒：

- ✅ **AAVS 配啱卡** = 每年免費 HK$200-400 cashback。低處果實。
- ⚠️ **公私數分清** = 月尾慳半個鐘 + 報數準確（公司 audit 通過）。
- 🎯 **月尾 audit 唔係 budget app 上身** = 15 分鐘看 pattern + 1 個 cut 建議。可持續。

今晚就試：開八達通 app → screenshot 上月 transaction → 貼入 Section 4 prompt → 15 分鐘有 insight。下個月再做一次，pattern 就清楚。
