## 情境

你屋企人嘅數碼記憶散落喺：

- iCloud Photos（自動同步你部電話）
- Google Photos（伴侶用 Android）
- 一個 5 年前嘅外部硬碟（細路仔嘅 BB 相，未驗證仲讀唔讀得返）
- WhatsApp 媒體資料夾（累積落嚟嘅語音訊息同影片）
- Mac 桌面亂放、諗住「下次先整理」嘅照片暫存資料夾

問題：

- 你唔知**呢啲備份完整嗎**
- 如果 iCloud 出事，你會失去邊 5 年嘅記憶
- 外部硬碟 5 年冇插過電，可能已經損壞

呢類「我以為已經備份好」嘅設定，每個香港家庭都有。真正災難來襲嗰陣先會發現出事。

呢篇用 Cowork 每月**自動檢視**：確認重要檔案有複數備份，並標示缺失嘅部分。

> ⚠️ Cowork 仲係研究預覽版。涉及家庭照片，設定做**唯讀**模式（唔准 Cowork 自動刪除或移動檔案）。

## 跟住做

### 1. 列出你嘅備份來源

- **來源 A**：iCloud（你部電話同 Mac 自動同步）
- **來源 B**：Google Photos（伴侶 / 之前用嘅 Android）
- **來源 C**：外部硬碟（手動備份）
- **來源 D**：其他（NAS / 雲端服務）

每個來源要列明：位置、容量大小、整理架構。

### 2. 用呢個 prompt 設定

<PromptBlock label="完整 prompt — Setup family backup audit">
我想設定一個每月自動執行嘅家庭檔案備份檢視。流程：

**觸發時間：** 每月第一個星期日，10:00

**輸入來源：**

1. iCloud Photos library（路徑：`/Users/[user]/Pictures/Photos Library.photoslibrary`）
2. Google Photos 匯出檔案（我每月手動觸發匯出，擺喺 `~/Documents/google-photos-monthly/`）
3. 外部硬碟：`/Volumes/Family-Backup`
4. Mac 桌面 + `~/Pictures/` 裡面未分類嘅照片

**每月執行：**

**第一步：盤點每個來源**

- 按年份統計項目數量（照片 + 影片）
- 總共用咗嘅儲存空間
- 最後修改日期範圍

**第二步：跨來源比對**
基於照片中繼資料 (metadata)（拍攝日期、位置、裝置），比對出：

- 邊啲照片同時存在喺 2 個或以上來源（安全）
- 邊啲只存在喺 1 個來源（有風險）
- 邊啲只喺電腦桌面未備份（緊急）

**第三步：健康檢查**

- 外部硬碟：檢查最後存取日期，標示超過 6 個月冇動過嘅硬碟
- iCloud：同步狀態（最後更新時間）
- Google Photos：匯出完整度

**第四步：關鍵文件檢視**

- `~/Documents/critical/` 資料夾（我會將重要文件擺入呢度）：屋契、保險單、身分證副本、出世紙、結婚證書
- 檢查每個檔案有冇雲端複本 + 實體列印本
- 標示任何只得一個來源嘅檔案

**第五步：生成報告**
寄一封電郵畀我：

- 標題：「[月份] Family Backup Audit Report — [N] 個緊急標示」
- 內容：每個部分嘅總結 + 具體行動（「呢個禮拜要插返外部硬碟同同步」、「關鍵文件 X 仲未有雲端複本」）
- 附帶：趨勢圖表（歷史每月儲存空間增長）

**重要限制：**

- 絕對唔好自動刪除 / 移動 / 重新命名任何檔案
- 唔好分享家庭照片內容嘅具體敘述（只檢視中繼資料同數量）
- 任何讀取操作只需最基本範圍，唔好深入分析個別檔案內容

設定好嘅話請確認，然後排程。
</PromptBlock>

### 3. 第一次檢視同手動修補

第一次檢視通常會搵出 5-10 個緊急標示。用第一個星期採取行動：

- 插返外部硬碟，驗證係咪仲讀到
- 搵出「只喺電腦桌面」嘅照片 → 同步落 iCloud
- 列印實體副本留底（出世紙 / 結婚證書 / 戶口簿）

過咗幾個月之後，檢視報告就會變薄（代表系統已經穩定）。你每個月用 5 分鐘睇完報告，夜晚都可以訓得安穩。

### 4. 1 年之後做一次「災難演習」

模擬災難發生：「如果我部 Mac 今晚壞咗，重要檔案我要點樣還原？」

嘗試行一次還原流程（從 iCloud 同外部硬碟還原），驗證實際係咪可行。通常會發現 1-2 個「以為備份緊但其實冇包到」嘅檔案。

### 變化 2：年長家人嘅備份方案

<PromptBlock label="變化 2 — 老人家 photo / doc backup">
我要幫年長家人（70 歲以上）做家庭照片同重要文件嘅備份。佢哋自己整唔到。

請設計一個更簡單嘅工作流程：

1. 一個「統一收集箱」位置（佢哋將實體相片同文件交畀我）
2. 我用嚟掃瞄 / 數碼化嘅高效流程（用 iPhone 掃瞄 / Apple Notes 掃瞄）
3. 備份到雲端 + 一個傳承計劃（如果我唔在場，邊個屋企人知道點樣存取）
4. 一個「故事標籤」嘅輕量標記（等將來下一代知道邊張相係邊個）

呢個情況嘅代價好高（不可重來），所以要強調冗餘備份 (redundancy)。
</PromptBlock>

2. 建議備份設定（比一般家庭更穩健）

## 一個現實評估

冇「完美」嘅備份設定。目標唔係 100% 絕對安全，係「真係出事嗰陣，將損失控制喺幾個月之內，而唔係冇咗幾年嘅心血」。

每月 5 分鐘檢視 + 每年一次災難演習 = 用極低成本換取 99% 嘅保護。比起完全冇檢視、自欺欺人咁諗「我有 iCloud 就 OK 啦」，到真正出事嗰日就會一無所有。

Claude Cowork 嘅角色：將 5 分鐘嘅檢視工作變得毫不費力，唔需要依賴你嘅自律去手動執行。
