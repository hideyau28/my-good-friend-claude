## 3. 語氣守則

### 3.1 基調：街坊大姐姐講嘢，唔係教科書

- 寫法要似**一個識嘢嘅朋友坐低同你傾**，唔係社工報告
- 用「你」直接對話，唔好用「家長們」「讀者」
- 句式短、直接、有節奏感
- 容許語氣詞（「喎」「㗎」「啦」「嘅」），但唔好每句都加

### 3.2 中英混雜原則

- **非白名單嘅英文詞，一句入面唔好超過一個。**（白名單詞彙不計入此限，例如「用 Claude API 起草 Prompt」係容許嘅，因為全屬白名單。）
- 如果成句都係廣東話但夾住一個英文動詞，多數可以搵到廣東話替代
- 白名單名詞前後唔使加引號，自然咁寫入去
- 唔好用英文做動詞：❌「去 set」「try fix」 → ✅「去定」「試下處理」
- **豁免動詞：** 已完全融入香港口語嘅動詞如 Keep（keep 住）、Send、Share 容許保留，不需硬譯為「保持」或「傳送」。

### 3.3 敏感話題語氣

- 心理健康、自傷、自殺——用**溫和但直接**嘅講法
- 唔好美化（「去咗更好嘅地方」），亦唔好煽情
- 數字同熱線電話**永遠具體**，唔好抽象
- 「self-harm」用「自傷」；「suicide」用「自殺」——跟香港心理健康機構慣例

### 3.4 唔好做嘅嘢

- ❌ 逐字硬譯英文句式（「這是一個很好的方法去⋯⋯」）
- ❌ 用「的」代替「嘅」（除非刻意書面語段落）
- ❌ 用普通話慣用語代替廣東話（「沒問題」→「冇問題」）
- ❌ 寫 lorem ipsum / placeholder 英文文案
- ❌ 英文標點夾廣東話（用中文標點：「」、——、⋯⋯）

### 3.5 英文動詞與時態助詞嘅結合

喺書面語境，**切勿將英文動詞直接加上「咗、緊、過」等廣東話助詞**。呢種寫法口語常見，但寫出嚟會令句法鬆散。

- ❌ 錯誤示範：「已 cancel 嘅」、「我 decline 咗」
- ✅ 正確做法：直接轉用中文動詞（「已取消嘅」、「我已拒絕」）

### 3.6 嚴格防禦大陸用語

AI 好容易喺翻譯過程混入大陸用語，審稿時必須替換為香港慣用語：

- 禁用「視頻」，改用「影片」
- 禁用「屏幕」，改用「螢幕 / 熒幕」
- 禁用「質量」，改用「質素 / 品質」
- 禁用「激活」，改用「啟動 / 啟用」
- 禁用「給力、走心、靠譜」等內地網絡詞，改用對應嘅香港廣東話

## 4. 審稿 Checklist

每篇 MDX 發佈前逐項核對：

### 語言
- [ ] Frontmatter `title` 同 `description` 冇非白名單英文
- [ ] 正文每段非白名單英文唔超過 2 個
- [ ] 冇英文動詞做謂語（set、fix、optimize、dump 等；Keep / Send / Share 屬豁免）
- [ ] 冇英文形容詞做定語（functional、vague、cautious 等）
- [ ] 冇英文動詞硬加助詞（「cancel 咗」「decline 咗」→ 轉中文動詞）
- [ ] 冇大陸用語（視頻、屏幕、質量、激活、給力 等）
- [ ] PromptBlock 內文同正文一致標準

### 語氣
- [ ] 讀出嚟似朋友傾偈，唔似翻譯文件
- [ ] 敏感話題用溫和直接語氣
- [ ] 冇過度美化 / 煽情

### 技術
- [ ] Frontmatter key（slug、category、section）冇改動
- [ ] PromptBlock label 冇用 `{...}` JSX expression
- [ ] URL / import / component name 冇改動
- [ ] MDX build 通過（`pnpm build` 或至少 `pnpm typecheck`）

### SEO
- [ ] `keywords` 陣列保留必要英文搜尋詞
- [ ] `description` 保留 DSE / JUPAS / Claude 等 SEO 關鍵字
- [ ] `title` 保持搜尋引擎友好長度（≤60 字）
